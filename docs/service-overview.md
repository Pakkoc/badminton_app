# 거트알림 - 서비스 소개

## 한 줄 요약

배드민턴 거트 작업의 접수부터 완료까지 실시간 진행 현황을 알려주어, 고객의 헛걸음을 방지하고 샵의 고객 관리를 돕는 모바일 앱.

---

## 어떤 문제를 해결하나요?

배드민턴 거트(스트링) 교체를 위해 샵에 라켓을 맡기면, "언제까지 해드릴게요"라는 구두 약속만 이루어진다. 실제 완료 시점을 정확히 알 수 없어 다음과 같은 문제가 반복된다.

- 약속 시간에 맞춰 갔지만 작업이 아직 완료되지 않아 **헛걸음하는 사례가 빈번**하다.
- 고객은 작업 진행 상황을 알 수 없어 **불안감과 불편함**을 느낀다.
- 샵은 고객 문의 전화를 반복적으로 받는 **운영 비효율**이 발생한다.

거트알림은 이 간극을 메운다.

- 사장님이 작업 상태를 변경하면 고객에게 **푸시 알림이 즉시 전송**된다.
- 고객은 앱에서 **작업 진행 현황을 실시간으로 확인**할 수 있다.
- 완료 알림을 받으면 **샵 위치 길찾기**로 바로 수령하러 갈 수 있다.

---

## 사용자는 누구인가요?

### 고객 (배드민턴 동호인)

거트 교체를 맡기고 완료 여부가 궁금한 배드민턴 동호인. 실시간 알림으로 헛걸음 없이 라켓을 수령할 수 있다.

### 샵 사장님 (거트 전문점 운영자)

거트 작업을 접수하고 관리하는 샵 운영자. 앱으로 작업 상태를 관리하여 문의 전화를 줄이고, 회원 관리를 통해 단골 고객을 확보할 수 있다.

---

## 서비스 이용 흐름

### 고객의 흐름

```
1. 오프라인에서 샵에 라켓을 맡긴다.
2. 사장님이 작업을 접수하면 "접수됨" 푸시 알림을 받는다.
3. 앱에서 작업 진행 현황(접수됨 → 작업중 → 완료)을 실시간으로 확인한다.
4. "완료" 푸시 알림을 받으면 샵 위치 길찾기로 라켓을 수령하러 간다.
```

### 샵 사장님의 흐름

```
1. 회원가입 시 "샵 사장님"을 선택하고 샵 정보를 등록한다.
2. 고객이 라켓을 맡기면 QR 스캔 또는 수동으로 회원을 등록한다.
3. 거트, 텐션, 라켓 정보를 입력하고 작업을 접수한다.
4. 작업 진행에 따라 상태를 변경한다. (접수됨 → 작업중 → 완료)
5. 상태 변경 즉시 고객에게 푸시 알림이 자동 전송된다.
```

---

## 핵심 차별점: QR코드 기반 간편 회원 등록

기존 샵에서는 고객 정보를 수기로 메모하거나, 별도의 관리 수단이 없는 경우가 대부분이다. 거트알림은 **QR코드 스캔 한 번으로 회원 등록과 작업 접수를 연결**하는 방식으로 이 과정을 혁신한다.

### 앱 사용 고객의 경우

고객이 "내 QR코드" 화면을 보여주면, 사장님이 스캔하여 이름과 연락처가 자동 입력된다. 선호 거트, 텐션, 라켓 정보만 추가하면 바로 작업 접수가 가능하다.

### 앱 미가입 고객의 경우

사장님이 수동으로 회원을 등록한다. 나중에 고객이 앱에 가입하면, 전화번호 기준으로 기존 회원 정보와 자동 매칭되어 이전 작업 이력까지 바로 확인할 수 있다.

## 핵심 기능

### 실시간 작업 현황 확인

고객은 앱에서 자신의 거트 작업 상태(접수됨 / 작업중 / 완료)를 실시간으로 확인할 수 있다. 각 상태 변경 시점이 타임라인으로 표시된다.

### 푸시 알림

사장님이 작업 상태를 변경하면 고객에게 즉시 푸시 알림이 전송된다. 완료 알림에는 샵 위치 바로가기가 포함되어, 탭 한 번으로 길찾기 화면으로 이동할 수 있다.

### 주변 샵 검색

네이버 지도에서 현재 위치 기준으로 주변 거트 샵을 검색할 수 있다. 지도 마커 또는 리스트 뷰로 확인 가능하며, 각 샵의 현재 작업 현황(접수 N건 / 작업중 N건)도 표시된다.

### QR코드 회원 등록

고객이 QR코드를 보여주면 사장님이 스캔하여 간편하게 회원 등록을 할 수 있다. 개인정보 최소화를 위해 QR에는 user_id만 인코딩된다.

### 회원 관리

사장님은 회원별 선호 거트, 텐션, 라켓 정보, 방문 이력을 체계적으로 관리할 수 있다. 단골 고객의 선호 정보를 미리 파악하여 빠른 작업 접수가 가능하다.

### 작업 관리 대시보드

사장님은 오늘의 작업 현황(접수 / 진행 / 완료 건수)을 한눈에 확인하고, 한 탭으로 작업 상태를 전환할 수 있다.

| 구분 | 비용 |
|------|------|
| 고객 - 앱 사용 | 무료 |
| 고객 - 작업 현황 확인 및 알림 | 무료 |

초기에는 모든 기능을 무료로 제공하여 사용자를 확보한다. 향후 샵 홍보 기능, 프리미엄 회원 관리 도구 등 부가 서비스를 통한 수익 모델을 검토할 수 있다.

## 주요 페이지 구성

| 페이지 | 설명 |
|--------|------|
| 스플래시 / 로그인 | 앱 진입, 역할 선택(고객 / 샵 사장님) 후 회원가입 |
| 고객 홈 | 현재 진행 중인 내 작업 목록 (상태 표시: 접수됨/작업중/완료) |
| 작업 상세 | 거트, 텐션, 라켓, 샵 정보, 상태 타임라인 |
| 주변 샵 검색 | 네이버 지도에 주변 샵 마커 표시, 리스트 뷰 전환 |
| 샵 상세 | 샵 소개, 위치, 연락처, 길찾기 버튼, 현재 작업 현황 |
| 내 QR코드 | 사장님에게 보여줄 QR코드 화면 |
| 작업 이력 | 과거 완료된 작업 목록 |
| 마이페이지 | 프로필 수정, 알림 설정 |
| 사장님 대시보드 | 오늘의 작업 현황, 최근 작업 목록 |
| 작업 접수 | QR 스캔 or 회원 검색 → 거트/텐션/라켓 입력 → 접수 |
| 작업 관리 | 전체 작업 목록, 상태 변경 버튼 |
| 회원 관리 | 회원 목록, 검색, 상세 정보 (선호 거트, 라켓, 방문 이력) |
| 샵 설정 | 샵 정보 수정, 사장님 개인정보 수정 |

## 기술 구성 요약

| 영역 | 기술 | 역할 |
|------|------|------|
| 프론트엔드 | Flutter | iOS/Android 크로스 플랫폼 앱 |
| DB/인증 | Supabase (PostgreSQL + Auth) | 데이터 저장, 사용자 인증 |
| 푸시 알림 | Firebase Cloud Messaging (FCM) | 작업 상태 변경 시 알림 전송 |
| 알림 트리거 | Supabase Edge Function | DB 트리거 → FCM 호출 |
| 지도 | Naver Map API | 주변 샵 검색, 길찾기 연동 |

---

## 핵심 정책

- **앱 미가입 고객 지원**: 고객이 앱에 가입하지 않아도 사장님이 수동으로 회원 등록 가능. 나중에 가입하면 전화번호로 자동 매칭
- **개인정보 최소화**: QR코드에는 user_id만 인코딩하여 개인정보 노출을 최소화
- **가입 시 역할 고정**: 고객과 샵 사장님 간 역할 전환은 불가
- **작업 정보 독립 저장**: 매 작업마다 거트, 텐션, 라켓 정보를 별도 저장하여 회원 선호값과 독립적으로 기록
