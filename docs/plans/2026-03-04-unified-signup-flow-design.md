# 통합 회원가입 플로우 설계

> 작성일: 2026-03-04

## 1. 배경 및 목적

현재 회원가입 시 고객/사장님 역할을 선택하며, 역할 변경이 불가능하다. 사장님은 고객 기능을 사용할 수 없다.

**변경 목표:** 모든 사용자가 고객으로 가입하고, 나중에 샵 사장님으로 추가 등록할 수 있게 한다. 사장님도 고객 기능을 자유롭게 사용할 수 있다.

## 2. 변경 전/후 플로우 비교

### 현재 플로우
```
소셜 로그인 → 프로필 설정 (역할 선택 + 이름 + 전화번호)
  ├─ 고객 선택 → /customer/home
  └─ 사장님 선택 → 샵 등록 → /owner/dashboard
역할 변경 불가. 사장님은 고객 기능 사용 불가.
```

### 변경 후 플로우
```
소셜 로그인 → 프로필 설정 (이름 + 전화번호, 역할 선택 없음)
  → /customer/home (모든 사용자 고객으로 시작)

나중에: 마이페이지 → "샵 사장님 등록" → 샵 등록 화면
  → 등록 완료 후 마이페이지에 "사장님 모드 전환" 메뉴 활성화
  → 전환 시 /owner/dashboard (사장님 하단탭)
  → 사장님 설정에서 "고객 모드 전환" 가능
```

## 3. 영향 범위

| 영역 | 변경 내용 |
|------|----------|
| 프로필 설정 화면 | 역할 선택 UI 제거, 스텝 인디케이터 제거 |
| 라우터 | 초기 진입 항상 고객 홈, 모드 전환 라우팅 추가 |
| 마이페이지 | "샵 사장님 등록" / "사장님 모드 전환" 메뉴 추가 |
| 사장님 설정 화면 | "고객 모드 전환" 메뉴 추가 |
| DB 마이그레이션 | 기존 `shop_owner` → `customer`로 변경 |
| Auth Provider | shops 테이블 조인으로 사장님 여부 판단 |
| Splash | 역할 기반 분기 제거, 항상 고객 홈으로 |
| Pencil 디자인 | 프로필 설정, 마이페이지 디자인 수정 |
| UI 스펙 | profile-setup, mypage 관련 스펙 수정 |

## 4. 화면별 상세 변경

### 4.1 프로필 설정 화면 (`/profile-setup`)

**제거:**
- 역할 선택 카드 UI (고객/사장님 2개 카드)
- 스텝 인디케이터 (2단계 표시)
- "가입 후 역할을 변경할 수 없습니다" 경고

**유지:**
- 이름 입력 (소셜 프로필에서 프리필)
- 전화번호 입력
- 회원 매칭 로직 (`matchMembersByPhone`)

**변경:**
- 버튼 텍스트: "시작하기" 통일
- 제출 시: `role = 'customer'` 고정, 항상 `/customer/home`으로 이동

### 4.2 마이페이지 (`/mypage`)

**추가 메뉴:**

| 조건 | 메뉴 항목 | 아이콘 | 동작 |
|------|----------|--------|------|
| 샵 미등록 | "샵 사장님 등록" | `storefront` | `/shop-register`로 이동 |
| 샵 등록 완료 | "사장님 모드 전환" | `swap_horiz` | `/owner/dashboard`로 이동 |

### 4.3 사장님 설정 화면 (`/owner/shop-settings`)

**추가 메뉴:**

| 메뉴 항목 | 아이콘 | 동작 |
|----------|--------|------|
| "고객 모드 전환" | `swap_horiz` | `/customer/home`으로 이동 |

### 4.4 라우터 변경

**Splash 분기 단순화:**
```
세션 없음 → /login
세션 있음 + 프로필 없음 → /profile-setup
세션 있음 + 프로필 있음 → /customer/home (항상)
```

**샵 등록 라우트:**
- `/shop-register`는 로그인된 모든 사용자가 접근 가능
- 이미 샵이 있으면 `/owner/dashboard`로 리다이렉트

## 5. Provider 변경

### 새 Provider
```dart
// 현재 유저가 샵을 보유하는지 여부
final hasShopProvider = FutureProvider<bool>(...);

// 현재 활성 모드
final activeModeProvider = StateProvider<AppMode>((ref) => AppMode.customer);

enum AppMode { customer, owner }
```

### 기존 변경
- `userRoleProvider`: 삭제 또는 deprecated → `hasShopProvider`로 대체
- `splashRouteProvider`: 항상 `/customer/home` 반환하도록 단순화

## 6. 모드 전환 UX

**고객 → 사장님:**
1. 마이페이지 → "사장님 모드 전환" 탭
2. 하단 네비 `[대시보드 | 작업관리 | 설정]`으로 변경
3. `/owner/dashboard`로 이동

**사장님 → 고객:**
1. 사장님 설정 → "고객 모드 전환" 탭
2. 하단 네비 `[홈 | 작업이력 | 마이페이지]`로 변경
3. `/customer/home`으로 이동

**규칙:**
- 앱 시작 시 항상 고객 모드
- 전환 시 `activeModeProvider`만 변경
- 별도 persist 불필요

## 7. DB 마이그레이션

```sql
-- 기존 shop_owner 사용자를 customer로 변경
UPDATE users SET role = 'customer' WHERE role = 'shop_owner';
```

`role` 컬럼은 유지하되 모든 사용자가 `customer` 값을 가진다. 사장님 여부는 `shops` 테이블 존재로 판단.

## 8. 에러 케이스

| 상황 | 처리 |
|------|------|
| 샵 미등록 상태에서 사장님 모드 전환 시도 | 불가능 (메뉴가 "샵 사장님 등록"으로 표시) |
| 샵 등록 중 이탈 | 미완료 상태, 마이페이지에서 "샵 사장님 등록" 메뉴 유지 |
| 사장님 모드에서 샵 삭제 시 | 고객 모드로 자동 전환 |

## 9. 접근법 선택 근거

**접근법 A 채택:** `role` 필드 유지 + `shops` 테이블로 사장님 판단
- DB 스키마 변경 최소화
- 기존 RLS 정책 대부분 유지 가능
- `role` 컬럼은 향후 다른 역할 확장 가능성 보존
