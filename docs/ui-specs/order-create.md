## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-order-create` |
| **화면 명** | 작업 접수 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| QR 사용자 정보 조회 | `users` | `id = QR의_user_id` | name, phone | QR 스캔 성공 시 |
| QR 기존 회원 확인 | `members` | `shop_id = 내_샵_id` AND `user_id = QR의_user_id` | member.* | QR 스캔 성공 시 |
| 회원 검색 | `members` | `shop_id = 내_샵_id` AND (`name ILIKE '%키워드%'` OR `phone ILIKE '%키워드%'`) | member.name, phone | 검색 입력 시 (debounce 300ms) |

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| QR 회원 자동 등록 | `members` | INSERT | `{shop_id, user_id, name, phone}` | QR 스캔으로 신규 회원일 때 |
| 작업 접수 | `orders` | INSERT | `{shop_id, member_id, memo, status: 'received'}` | "작업 접수하기" 버튼 탭 시 |

**Supabase 쿼리 예시:**

```dart
// QR 스캔 후 회원 확인/등록
final user = await supabase
  .from('users')
  .select('id, name, phone')
  .eq('id', scannedUserId)
  .single();

final existingMember = await supabase
  .from('members')
  .select()
  .eq('shop_id', myShopId)
  .eq('user_id', scannedUserId)
  .maybeSingle();

if (existingMember == null) {
  // 신규 회원 등록
  await supabase.from('members').insert({
    'shop_id': myShopId,
    'user_id': scannedUserId,
    'name': user['name'],
    'phone': user['phone'],
    'visit_count': 1,
  });
}

// 작업 접수
await supabase.from('orders').insert({
  'shop_id': myShopId,
  'member_id': memberId,
  'memo': memo,
  'status': 'received',
});
```

## 9. 비고

- **QR 스캔이 주요 방법**: QR 스캔 영역을 화면 최상단에 크게 배치하여 주요 접수 동선임을 시각적으로 강조한다. 버튼 크기를 일반 버튼보다 크게(52px) 설정한다.
- **간편 접수 흐름**: QR 스캔 → 회원 자동 확인 → 메모 입력(선택) → 바로 접수. 최소한의 입력으로 빠르게 접수할 수 있도록 한다.
- **카메라 권한**: QR 스캔 시 카메라 권한이 없으면 권한 요청 다이얼로그 표시. 거부 시 "카메라 권한이 필요합니다" 안내와 함께 설정 이동 링크 제공.
- **회원 상세에서 진입 시**: `member_id`가 전달되면 QR/검색 단계를 건너뛰고 바로 회원 정보 카드 + 작업 폼을 표시한다.
- **키보드 처리**: 입력 폼이 키보드에 가려지지 않도록 `SingleChildScrollView`로 감싼다.
- **접수 완료 후**: 성공 시 대시보드로 이동하며, 연속 접수를 위해 토스트에 "연속 접수" 버튼을 3초간 표시한다. 탭 시 빈 작업 접수 화면으로 재진입.
- **텍스트 명암비**: 본문 텍스트(`#475569`)와 배경(`#FFFFFF`)의 명암비는 WCAG AA 기준 4.5:1 이상을 충족해야 한다.
- **터치 타겟**: 모든 터치 가능 요소(QR 스캔 버튼, 접수 버튼, 검색 결과 항목, 변경 버튼)의 터치 영역은 최소 44x44px을 보장한다. 인접 터치 타겟 간 최소 8px 간격을 유지한다.
