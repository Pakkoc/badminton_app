# 사장님 대시보드 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-dashboard` |
| **화면 명** | 사장님 대시보드 |
| **목적** | 오늘의 작업 현황(접수/작업중/완료 건수)과 최근 작업 목록을 한눈에 파악하여 샵 운영 상황을 빠르게 확인한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`인 사용자만 접근 가능, 샵 등록 완료 상태 |

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | "거트알림" 타이틀과 샵 이름 표시 | 배경 `#FFFFFF`, 하단 테두리 `#E2E8F0` 0.5px |

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| received_count | int | orders COUNT | 숫자 (32sp) | 오늘 접수됨 상태 건수 |
| in_progress_count | int | orders COUNT | 숫자 (32sp) | 오늘 작업중 상태 건수 |
| completed_count | int | orders COUNT | 숫자 (32sp) | 오늘 완료 상태 건수 |
| member_name | String | members.name | 텍스트 | 회원 이름 |
| gut_name | String | orders.gut_name | 텍스트 | 거트명 |
| tension | String | orders.tension | 텍스트 (예: "25lbs") | 텐션값 |
| racket_model | String | orders.racket_model | 텍스트 | 라켓 모델명 |
| status | Enum | orders.status | StatusBadge | 작업 상태 |
| created_at | Timestamp | orders.created_at | "HH:mm 접수" | 접수 시간 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 오늘 상태별 건수 조회 | `orders` | `shop_id = 내_샵_id` AND `created_at >= 오늘_00:00` | COUNT by status | 화면 진입 시, Pull-to-refresh 시 |
| 최근 작업 목록 조회 | `orders` JOIN `members` | `shop_id = 내_샵_id` AND `created_at >= 오늘_00:00` ORDER BY `created_at` DESC LIMIT 5 | order.*, member.name | 화면 진입 시, Pull-to-refresh 시 |

**Supabase 쿼리 예시:**

```dart
// 오늘 상태별 건수
final today = DateTime.now().toIso8601String().substring(0, 10);
final counts = await supabase
  .from('orders')
  .select('status')
  .eq('shop_id', myShopId)
  .gte('created_at', '${today}T00:00:00');

// 최근 작업 목록
final recentOrders = await supabase
  .from('orders')
  .select('*, members(name, phone)')
  .eq('shop_id', myShopId)
  .gte('created_at', '${today}T00:00:00')
  .order('created_at', ascending: false)
  .limit(5);
```

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 작업 상태 변경 | `orders` | UPDATE | `{status: 새_상태, updated_at: now()}` | 상태 변경 버튼 탭 시 |

### 6.3 실시간 구독

| 구독 대상 | 테이블 | 조건 | 이벤트 | 처리 |
|-----------|--------|------|--------|------|
| 내 샵 작업 변경 | `orders` | `shop_id = 내_샵_id` | INSERT, UPDATE | 카운트 숫자 재계산, 최근 작업 목록 갱신 |

## 9. 비고

- 오늘의 작업 현황 카드와 최근 작업 목록이 **균등한 비중**으로 화면을 차지하도록 설계한다. 현황 카드가 화면 상단 약 40%, 최근 작업 목록이 하단 약 60%를 차지한다.
- 카운트 카드는 탭 가능하며, 탭 시 해당 상태로 필터된 작업관리 화면으로 이동한다. 탭 시 살짝 눌림(scale 0.98) 애니메이션을 적용한다.
- 상태 변경 버튼 탭 시 확인 다이얼로그 없이 즉시 변경한다 (빠른 운영을 위해). 단, 변경 후 3초간 "실행 취소" 스낵바를 표시하여 오조작을 방지한다.
- 최근 작업 목록은 오늘 날짜 기준이며 최대 5건만 표시한다. 5건 초과 시 "전체보기" 링크로 작업관리 화면 유도.
- FAB는 스크롤 시 잠시 축소되었다가 스크롤 멈추면 다시 표시되는 동작을 적용한다.
- 디자인 참고: 토스의 카드 기반 대시보드, 배민의 둥근 UI 스타일.
- **텍스트 명암비**: 본문 텍스트(`#475569`)와 배경(`#FFFFFF`)의 명암비는 WCAG AA 기준 4.5:1 이상을 충족해야 한다. 보조 텍스트(`#94A3B8`)는 `bodySmall`(12sp) 이상에서만 사용한다.
- **터치 타겟**: 모든 터치 가능 요소(카운트 카드, 작업 카드, FAB, 상태 변경 버튼, 하단 네비게이션)의 터치 영역은 최소 44x44px을 보장한다. 인접 버튼 간 최소 8px 간격을 유지한다.
