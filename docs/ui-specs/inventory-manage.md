# 재고 관리 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-inventory-manage` |
| **화면 명** | 재고 관리 |
| **목적** | 사장님이 샵의 상품 재고(거트, 라켓 등)를 등록/수정/삭제하여 관리하고, 고객이 샵 상세 화면에서 재고를 열람할 수 있도록 한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`, 샵 등록 완료 상태 |

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 버튼 + "재고 관리" 타이틀 | 배경 `#FFFFFF` |
| 상품 목록 | 상품 그리드 (이미지, 상품명, 카테고리, 재고 수량) | 2열 그리드, 스크롤 가능 |

**바텀시트 스타일:**

| 속성 | 값 |
|------|-----|
| 배경색 | `#FFFFFF` |
| 모서리 | 상단 좌우 16px |

| 시트 타이틀 | `titleLarge` (18sp, SemiBold), `#0F172A` |
| 필드 간 간격 | 16px |
| 저장 버튼 배경 | `#F97316` (활성) / `#E2E8F0` (비활성) |
| 저장 버튼 텍스트 | `#FFFFFF`, 16sp, SemiBold |
| 저장 버튼 높이 | 52px |
| 저장 버튼 모서리 | 12px |
| 삭제 버튼 텍스트 | `#EF4444`, 14sp |

**카테고리 입력 동작:**
- 고정 카테고리 목록을 드롭다운으로 제공: 라켓, 상의, 하의, 가방, 신발, 악세서리
- 자유 입력은 불가, 목록에서만 선택

**이미지 영역 동작:**
- 탭 시 카메라/갤러리 선택 바텀시트 표시
- 이미지가 등록되면 썸네일로 표시 (64x64px, 모서리 8px, `BoxFit.cover`)
- 이미지 썸네일 우측 상단에 삭제 버튼 표시: `close` 아이콘 12px, 원형 배경 `rgba(0,0,0,0.6)` 20x20px, 아이콘 색상 `#FFFFFF`
- 삭제 버튼 탭 시 이미지 즉시 제거
- 이미지가 없으면 "이미지 추가" 안내 + 카메라 아이콘 표시

## 4. 필드 정의

### 4.1 표시 필드 (그리드)

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| product_name | String | inventory.name | 텍스트 그대로 | 상품명 |
| category | String | inventory.category | 텍스트 그대로 | 상품 카테고리 |

### 4.2 입력 필드 (바텀시트)

| 필드명 | 타입 | 필수 | 기본값 | 검증 규칙 | 설명 |
|--------|------|------|--------|-----------|------|
| name | String | Y | "" | 1~50자, 공백만 불가 | 상품명 |
| category | String (enum) | Y | "" | 라켓/상의/하의/가방/신발/악세서리 중 택1 | 상품 카테고리 |
| quantity | Int | Y | 0 | 0~9999, 정수 | 재고 수량 |
| image | File | N | null | 10MB 이하 | 상품 이미지 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 재고 목록 조회 | `inventory` | `shop_id = 내_샵_id` ORDER BY `name` ASC | inventory.* | 화면 진입 시 |
| 카테고리 목록 조회 | `inventory` | `shop_id = 내_샵_id` DISTINCT `category` | category 목록 | 바텀시트 표시 시 |

**Supabase 쿼리 예시:**

```dart
// 재고 목록 조회
final items = await supabase
  .from('inventory')
  .select('*')
  .eq('shop_id', myShopId)
  .order('name', ascending: true);

// 카테고리 목록 조회
final categories = await supabase
  .from('inventory')
  .select('category')
  .eq('shop_id', myShopId)
  .not('category', 'is', null);
// 클라이언트에서 중복 제거
```

### 6.2 데이터 변경

| API | 테이블/서비스 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 이미지 업로드 | Supabase Storage | UPLOAD | 이미지 파일 | 저장 버튼 탭 시 (이미지 변경 시) |
| 상품 추가 | `inventory` | INSERT | `{shop_id, name, category, quantity, image_url}` | 추가 바텀시트 "저장" 탭 시 |
| 상품 수정 | `inventory` | UPDATE | `{name, category, quantity, image_url}` | 수정 바텀시트 "저장" 탭 시 |
| 상품 삭제 | `inventory` | DELETE | `id` | 삭제 확인 다이얼로그에서 "삭제" 탭 시 |

**Supabase 쿼리 예시:**

```dart
// 이미지 업로드
String? imageUrl;
if (imageFile != null) {
  final path = 'inventory/${myShopId}/${DateTime.now().millisecondsSinceEpoch}.jpg';
  await supabase.storage.from('inventory-images').upload(path, imageFile);
  imageUrl = supabase.storage.from('inventory-images').getPublicUrl(path);
}

// 상품 추가
await supabase.from('inventory').insert({
  'shop_id': myShopId,
  'name': productName,
  'category': category.isNotEmpty ? category : null,
  'quantity': quantity,
  'image_url': imageUrl,
});

// 상품 수정
await supabase.from('inventory').update({
  'name': productName,
  'category': category.isNotEmpty ? category : null,
  'quantity': quantity,
  'image_url': imageUrl,
}).eq('id', itemId);

// 상품 삭제
await supabase.from('inventory').delete().eq('id', itemId);
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 화면 최초 진입 시 데이터 로드 중 | 스켈레톤 shimmer 표시 (그리드 형태) |
| **빈 상태** | 등록된 상품 0건 | 빈 상태 아이콘 + "등록된 상품이 없습니다" 안내 |
| **정상** | 상품이 1건 이상 존재 | 상품 그리드 표시 |
| **에러** | 네트워크 오류 또는 서버 오류 | "데이터를 불러올 수 없습니다" + 재시도 버튼 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |

| `hasError` | bool | false | 에러 발생 여부 |

| `productCategory` | String | "" | 바텀시트 카테고리 입력값 |
| `productQuantity` | int | 0 | 바텀시트 재고 수량 입력값 |
| `productImage` | File? | null | 바텀시트 이미지 |
| `existingCategories` | List\<String\> | [] | 기존 카테고리 목록 (드롭다운용) |
| `isSaving` | bool | false | 저장 중 여부 |

## 9. 비고

- **열람 전용 정책**: 고객은 샵 상세 화면의 "가게 재고" 탭에서 재고를 열람만 할 수 있으며, 구매 기능은 제공하지 않는다. 재고 관리(추가/수정/삭제)는 사장님만 가능하다.
- **기본 이미지**: 상품 이미지가 등록되지 않은 경우, 기본 아이콘(`inventory_2`)을 배경 `#F1F5F9` 위에 표시한다.
- **카테고리 자유 입력**: 카테고리는 기존에 등록된 카테고리 목록을 드롭다운으로 제공하되, 사장님이 새로운 카테고리를 자유롭게 입력할 수도 있다.
- **이미지 압축**: 업로드 전 이미지를 적절한 크기로 압축하여 저장 용량을 절약한다.
- **FAB 동작**: 스크롤 시 잠시 축소되었다가 스크롤 멈추면 다시 표시되는 동작을 적용한다 (대시보드와 동일).
- **삭제 안전장치**: 상품 삭제 시 항상 확인 다이얼로그를 표시하여 오조작을 방지한다.

### 접근성
- 뒤로가기 아이콘 버튼에 접근성 라벨(`semanticLabel: "뒤로가기"`)을 포함한다.
- FAB에 접근성 라벨(`semanticLabel: "상품 추가"`)을 포함한다.
- 하단 네비게이션 아이콘에 접근성 라벨을 포함한다 (각 탭명과 동일).
- 바텀시트 이미지 추가 영역에 접근성 라벨(`semanticLabel: "상품 이미지 추가"`)을 포함한다.
- 기본 이미지 아이콘은 색상만이 아닌 아이콘 형태로 "이미지 없음"을 표현하여, 색상만으로 정보를 구분하지 않는다.

### 피드백 보완
- 저장 버튼 탭 시 버튼에 로딩 상태를 표시하고, 중복 탭을 방지한다 (`isSaving` 플래그 활용).
- 저장 성공 시 "저장되었습니다" 토스트, 삭제 성공 시 "삭제되었습니다" 토스트, 실패 시 에러 스낵바를 표시한다.
- 바텀시트 입력 검증 에러는 해당 필드 바로 아래에 에러 메시지를 표시한다.

### Flutter 구현 참고
- 목록 화면은 `GridView.builder`를 사용하여 상품 그리드를 렌더링한다.
- 바텀시트 폼은 `Form` + `GlobalKey<FormState>`로 유효성 검증을 수행한다.
- 재고 수량 입력에는 `keyboardType: TextInputType.number`를 적용한다.
