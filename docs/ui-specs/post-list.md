# 게시글 목록 — UI 화면 스펙

> 최종 수정일: 2026-02-27

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-post-list` |
| **화면 명** | 게시글 목록 |
| **목적** | 샵의 공지사항과 이벤트 게시글을 카테고리별로 탐색한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, `shop_id` 필수 전달 |

---

## 2. 레이아웃 구조

```
┌─────────────────────────────┐
│  상단 앱바                    │  56px

├─────────────────────────────┤
│  카테고리 탭                   │  44px
│  [공지사항]  [이벤트]          │
├─────────────────────────────┤
│                             │
│  게시글 카드 리스트 (스크롤)     │
│  ┌─────────────────────────┐│
│  │ 공지사항  ·  2026-02-20  ││
│  │ 7월 휴무 안내            ││
│  │ 7월 15일부터 18일까지... ││
│  └─────────────────────────┘│
│        12px 간격             │
│  ┌─────────────────────────┐│
│  │ 공지사항  ·  2026-02-15  ││
│  │ 영업시간 변경 안내        ││
│  │ 3월부터 영업시간이...    ││
│  └─────────────────────────┘│
│        ...                  │
│                             │
├─────────────────────────────┤
```

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 + "게시판" 타이틀 | 배경 `#FFFFFF`, 하단 테두리 `#E2E8F0` 0.5px |
| 카테고리 탭 | 공지사항 / 이벤트 2개 탭 | 활성 탭: `#22C55E`, 비활성: `#94A3B8` |
| 게시글 카드 리스트 | 카테고리별 게시글 카드 목록 | gap 12px, padding 16px |
| 하단 네비게이션 | 고객용 5탭 | 비활성 상태 (게시판은 별도 탭 없음) |

## 3. 컴포넌트 목록

### 3.1 상단 앱바

| 컴포넌트 | 타입 | 설명 | 스타일 |
|----------|------|------|--------|
| 뒤로가기 | IconButton | 이전 화면으로 복귀 | Material Icon `arrow_back_ios`, 24px, `#1E293B` |
| 타이틀 | Text | "게시판" | `titleLarge` (20sp, Bold), `#1E293B` |

### 3.2 카테고리 탭

| 컴포넌트 | 타입 | 설명 | 스타일 |
|----------|------|------|--------|
| 공지사항 탭 | Tab | "공지사항" | 활성: `#22C55E` SemiBold, 비활성: `#94A3B8` |
| 이벤트 탭 | Tab | "이벤트" | 활성: `#22C55E` SemiBold, 비활성: `#94A3B8` |

**탭 바 스타일:**
- 배경: `#FFFFFF`
- 높이: 44px
- 인디케이터: `#22C55E`, 2px
- 텍스트: 14sp

### 3.3 게시글 카드

| 컴포넌트 | 타입 | 설명 | 스타일 |
|----------|------|------|--------|
| 카드 컨테이너 | Card | 게시글 카드 | 배경 `#FFFFFF`, cornerRadius 12px, stroke `#E2E8F0` 1px |
| 카테고리 뱃지 | Chip | "공지사항" 또는 "이벤트" | pill shape, 공지: 배경 `#F0FDF4` 텍스트 `#166534`, 이벤트: 배경 `#FEF3C7` 텍스트 `#92400E` |
| 날짜 | Text | 게시 날짜 | `bodySmall` (12sp), `#94A3B8` |
| 제목 | Text | 게시글 제목 | `titleSmall` (14sp, Bold), `#1E293B`, 최대 1줄 |
| 미리보기 | Text | 본문 앞부분 | `bodySmall` (13sp), `#475569`, 최대 2줄 |

**카드 레이아웃:**
- 내부 패딩: 16px
- 뱃지·날짜 Row: 간격 8px
- 뱃지 ~ 제목: 8px
- 제목 ~ 미리보기: 4px
- 카드 간 간격: 12px

### 3.4 빈 상태

- "등록된 게시글이 없습니다" (`bodyMedium`, 14sp, `#94A3B8`, 중앙 정렬)

## 4. 필드 정의

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| category | Enum | posts.category | 뱃지 | `notice` / `event` |
| title | String | posts.title | 텍스트, 최대 1줄 | 게시글 제목 |
| content | String | posts.content | 텍스트, 최대 2줄 미리보기 | 게시글 본문 |
| created_at | Timestamp | posts.created_at | "YYYY-MM-DD" 형식 | 게시 날짜 |
| image_url | String? | posts.image_url | 썸네일 (이벤트 카드) | 대표 이미지 |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 뒤로가기 | 앱바 뒤로가기 버튼 | 이전 화면으로 복귀 | 샵 상세로 돌아감 |
| 2 | 탭 전환 | 공지사항/이벤트 탭 터치 | 해당 카테고리 게시글 표시 | 탭 인디케이터 이동 |
| 3 | 게시글 카드 탭 | 카드 터치 | 게시글 상세로 이동 | `customer-post-detail` 화면 전환 |
| 4 | 하단 탭 전환 | 하단 네비게이션 탭 터치 | 해당 화면으로 이동 | 선택된 탭 화면 전환 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 게시글 목록 조회 | `posts` | `shop_id = shop_id` AND `category = 선택된_탭` ORDER BY `created_at` DESC | posts.* | 화면 진입 시, 탭 전환 시 |

---

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 데이터 로드 중 | 스켈레톤 shimmer (카드 3개) |
| **빈 상태** | 게시글 0건 | 빈 상태 텍스트 |
| **정상** | 게시글 1건 이상 | 카드 리스트 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `selectedTabIndex` | int | 0 | 선택된 탭 (0: 공지사항, 1: 이벤트) |
| `posts` | List\<Post\> | [] | 게시글 목록 |
| `isLoading` | bool | true | 로딩 중 여부 |

## 8. 네비게이션

### 8.1 진입 경로

| 출발 화면 | 동작 | 전달 파라미터 |
|-----------|------|---------------|
| 샵 상세 (`customer-shop-detail`) | 공지사항/이벤트 카드 탭 | `shop_id`, `category` (선택) |

### 8.2 이동 가능 화면

| 대상 화면 | 동작 | 전달 파라미터 |
|-----------|------|---------------|
| 게시글 상세 (`customer-post-detail`) | 게시글 카드 탭 | `post_id` |

---

## 9. 비고

- 기본 선택 탭은 진입 시 전달된 `category` 파라미터에 따라 결정된다. 없으면 "공지사항".
- 탭 전환 시 스와이프 제스처를 지원한다 (TabBarView).
- 하단 네비게이션을 포함하여 고객 흐름 내 다른 화면으로 이동 가능하다.

### Flutter 구현 참고
- `StatefulWidget` (탭 전환 + 데이터 상태 관리).
- `ListView.builder`로 게시글 목록 구현.
- `TabController`로 탭 전환 관리.
