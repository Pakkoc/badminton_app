# 관리자 샵 등록 요청 상세 — UI 화면 스펙

> 최종 수정일: 2026-03-12

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `admin-shop-request-detail` |
| **화면 명** | 샵 등록 요청 상세 |
| **목적** | 관리자가 개별 샵 등록 요청의 상세 정보를 검토하고, 승인 또는 거절 처리를 수행한다 |
| **사용자 역할** | admin |
| **진입 조건** | 로그인 필수, role이 `admin` |

---

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | "요청 상세" 타이틀 + 뒤로가기 | 배경 `#FFFFFF15` |
| 상태 배너 | 현재 샵 등록 요청 상태 표시 | 상태별 색상 분기 |
| 샵 정보 카드 | 샵 이름, 연락처, 주소, 사업자등록번호, 소개글 | 세부 정보 표시 |

| 액션 버튼 영역 | 거절/승인 버튼 | pending 상태일 때만 표시 |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 뒤로가기 | 뒤로가기 버튼 터치 | 목록 화면으로 복귀 | 목록 자동 갱신 |
| 2 | 승인 버튼 탭 | "승인" 버튼 터치 | 확인 다이얼로그 "이 샵을 승인하시겠습니까?" 표시 | 확인 시 shops UPDATE (status='approved', reviewed_at=now()) |
| 3 | 거절 버튼 탭 | "거절" 버튼 터치 | 거절 사유 입력 다이얼로그 표시 | 사유 입력 후 shops UPDATE (status='rejected', reject_reason, reviewed_at=now()) |
| 4 | 승인 확인 | 다이얼로그 "확인" 터치 | API 호출 | 성공: "승인 처리되었습니다" 토스트 + 목록으로 복귀 / 실패: 에러 스낵바 |
| 5 | 거절 확인 | 거절 다이얼로그 "거절" 터치 | API 호출 | 성공: "거절 처리되었습니다" 토스트 + 목록으로 복귀 / 실패: 에러 스낵바 |

### 승인 처리 흐름

```
승인 버튼 탭
  ↓
확인 다이얼로그 "이 샵을 승인하시겠습니까?"
  ↓
├── 취소 → 다이얼로그 닫힘
│
└── 확인
      ↓
    버튼 로딩 상태 전환
      ↓
    shops UPDATE (status='approved', reviewed_at=now())
      ↓
    ├── 성공 → "승인 처리되었습니다" 토스트 + 목록으로 복귀
    └── 실패 → 에러 스낵바 + 버튼 재활성화
```

### 거절 처리 흐름

```
거절 버튼 탭
  ↓
거절 사유 입력 다이얼로그
  ↓
├── 취소 → 다이얼로그 닫힘
│
└── 사유 입력 + "거절" 버튼 탭
      ↓
    버튼 로딩 상태 전환
      ↓
    shops UPDATE (status='rejected', reject_reason=입력값, reviewed_at=now())
      ↓
    ├── 성공 → "거절 처리되었습니다" 토스트 + 목록으로 복귀
    └── 실패 → 에러 스낵바 + 버튼 재활성화
```

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 샵 상세 조회 | `shops` | `id = shopId` | 전체 샵 정보 | 화면 진입 시 |
| 사장님 정보 조회 | `users` | `id = shops.owner_id` | name, phone, created_at | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
// 샵 상세 + 사장님 정보 조인 조회
final shop = await supabase
  .from('shops')
  .select('*, owner:users!owner_id(name, phone, created_at)')
  .eq('id', shopId)
  .single();
```

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 샵 승인 | `shops` | UPDATE | `{status: 'approved', reviewed_at: now()}` | 승인 확인 시 |
| 샵 거절 | `shops` | UPDATE | `{status: 'rejected', reject_reason: 사유, reviewed_at: now()}` | 거절 확인 시 |

**Supabase 쿼리 예시:**

```dart
// 승인
await supabase
  .from('shops')
  .update({
    'status': 'approved',
    'reviewed_at': DateTime.now().toIso8601String(),
  })
  .eq('id', shopId);

// 거절
await supabase
  .from('shops')
  .update({
    'status': 'rejected',
    'reject_reason': rejectReason,
    'reviewed_at': DateTime.now().toIso8601String(),
  })
  .eq('id', shopId);
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 화면 최초 진입 시 데이터 로드 중 | 전체 스켈레톤 shimmer |
| **에러** | 네트워크 오류 또는 서버 오류 | ErrorView "요청 정보를 불러올 수 없습니다" + 재시도 버튼 |
| **정상** | 데이터 로드 완료 | 전체 화면 표시, pending이면 액션 버튼 표시 |
| **처리 중** | 승인/거절 API 호출 중 | 해당 버튼에 로딩 인디케이터, 다른 버튼 비활성 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `isProcessing` | bool | false | 승인/거절 처리 중 여부 |
| `rejectReason` | String | "" | 거절 사유 입력값 (다이얼로그 내부) |

## 9. 비고

- 관리자(admin) 전용 화면으로, role이 `admin`인 사용자만 접근 가능하다.
- pending 상태가 아닌 요청(이미 승인/거절된 요청)은 상세 정보만 조회 가능하고, 액션 버튼은 표시하지 않는다.
- 거절 시 반드시 사유를 입력해야 한다. 사유는 사장님에게 마이페이지에서 안내된다.
- 승인 처리 시 확인 다이얼로그를 표시하여 오조작을 방지한다.

### 접근성

- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 거절 버튼의 빨간색(`#EF4444`)은 색상 + "거절" 텍스트로 구분한다.
- 지도 미리보기에 `Semantics(label: "샵 위치 지도 미리보기")` 를 적용한다.

### 성공/실패 피드백

- 승인 성공: "승인 처리되었습니다" 스낵바 (배경 `#10B981`, 텍스트 `#FFFFFF`)
- 거절 성공: "거절 처리되었습니다" 스낵바 (배경 `#F59E0B`, 텍스트 `#FFFFFF`)
- 처리 실패: "처리에 실패했습니다. 다시 시도해주세요" 스낵바 (배경 `#EF4444`, 텍스트 `#FFFFFF`)

### Flutter 구현 참고

- 이 화면은 승인/거절 처리 상태를 관리하므로 `StatefulWidget`으로 구현한다.
- 거절 사유 입력 다이얼로그는 `showDialog`로 구현하며, `Form` + `GlobalKey<FormState>`를 사용한다.
