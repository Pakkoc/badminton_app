# 회원 등록 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-member-register` |
| **화면 명** | 회원 등록 |
| **목적** | 앱 미가입 고객을 샵 회원으로 수동 등록하여 작업 접수와 고객 관리를 가능하게 한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`, 샵 등록 완료 상태 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 연락처 중복 확인 | `members` | `shop_id = 내_샵_id` AND `phone = 입력_연락처` | member.id, name | 연락처 11자리 입력 완료 시 |
| 최근 사용 거트 목록 | `orders` | `shop_id = 내_샵_id` GROUP BY `gut_name` ORDER BY COUNT DESC | gut_name (DISTINCT, 상위 10개) | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
// 연락처 중복 확인
final existing = await supabase
  .from('members')
  .select('id, name')
  .eq('shop_id', myShopId)
  .eq('phone', phoneNumber)
  .maybeSingle();

// 최근 사용 거트 목록
final guts = await supabase
  .from('orders')
  .select('gut_name')
  .eq('shop_id', myShopId)
  .order('created_at', ascending: false)
  .limit(100);
// 클라이언트에서 중복 제거 후 빈도순 정렬
```

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 회원 등록 | `members` | INSERT | `{shop_id, name, phone, preferred_gut, preferred_tension, racket_info, memo, visit_count: 0}` | "회원 등록하기" 버튼 탭 시 |

**Supabase 쿼리 예시:**

```dart
await supabase.from('members').insert({
  'shop_id': myShopId,
  'name': name,
  'phone': phone,
  'preferred_gut': preferredGut.isNotEmpty ? preferredGut : null,
  'preferred_tension': preferredTension.isNotEmpty ? preferredTension : null,
  'racket_info': racketList.where((r) => r.isNotEmpty).toList().isNotEmpty
    ? racketList.where((r) => r.isNotEmpty).map((r) => {'model': r}).toList()
    : null,
  'memo': memo.isNotEmpty ? memo : null,
  'visit_count': 0,
});
```

## 9. 비고

- **폼 그룹핑**: 기본 정보 / 선호 정보 / 라켓 정보 / 메모를 각각 카드로 그룹핑하여 시각적으로 구분한다. 각 카드 배경: `#FFFFFF`, 모서리: 16px, 패딩: 16px, 간격: 16px.
- **자동 하이픈**: 연락처 입력 시 3-4-4 형식으로 자동 하이픈을 삽입한다. 숫자만 입력 가능하도록 `TextInputType.phone` 적용.
- **연락처 중복 시**: 경고 UI를 표시하지만 등록을 차단하지는 않는다. 동명이인이나 가족 회원 등 동일 연락처로 등록이 필요한 경우가 있기 때문이다. 경고 메시지: "이 연락처로 등록된 회원이 있습니다: 홍길동" (배경 `#FEF3C7`, 텍스트 `#92400E`).
- **작업 접수에서 진입 시**: `returnTo: 'order-create'`가 전달되면 등록 완료 후 자동으로 작업 접수 화면으로 복귀하며, 등록된 회원이 자동 선택된 상태가 된다.
- **라켓 정보 저장 형식**: JSONB 형식으로 저장. 예: `[{"model": "YONEX AX99 Pro"}, {"model": "Victor DZS"}]`
- **키보드 처리**: 각 입력 필드 간 "다음" 버튼으로 포커스 이동. 마지막 필드에서 "완료" 버튼. 키보드에 가려지지 않도록 `SingleChildScrollView` + `padding` 적용.
- **필수 표시**: 필수 필드 라벨에 빨간색 "*" 표시 (`#EF4444`).
- **빈 라켓 처리**: 빈 문자열인 라켓 입력 행은 저장 시 자동으로 제외한다.
