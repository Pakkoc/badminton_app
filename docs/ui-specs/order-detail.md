# 작업 상세 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-order-detail` |
| **화면 명** | 작업 상세 |
| **목적** | 고객이 자신의 거트 작업 상태를 상세히 확인하고, 작업 정보(라켓, 거트, 텐션)와 상태 타임라인, 샵 정보를 한 화면에서 파악한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능. `order_id` 필수 전달 |

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 버튼 + "작업 상세" 타이틀 | 앱바 배경 `#FFFFFF`, 하단 테두리 `#E2E8F0` 0.5px |
| 상태 뱃지 (크게) | 현재 작업 상태를 화면 상단에 크게 표시 | 홈 화면 상태 뱃지의 확대 버전 |
| 작업 정보 섹션 | 라켓 모델명, 거트명, 텐션 정보를 아이콘과 함께 표시 | 카드 형태, 각 항목은 라벨 + 값 구조 |
| 진행 상태 타임라인 | 접수됨 → 작업중 → 완료 순서로 타임라인 표시 | 각 단계에 도달 시점을 표시. 미도달 단계는 비활성 처리 |
| 작업 메모 섹션 | 사장님이 작성한 작업 메모 표시 | 메모가 없으면 섹션 자체를 숨김 |
| 샵 정보 섹션 | 샵 이름, 주소, 연락처, 길찾기 버튼 | 길찾기 버튼은 네이버 지도 앱 연동 |

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| racket_model | String | orders.racket_model | 텍스트 그대로 | 라켓 모델명 |
| gut_name | String | orders.gut_name | 텍스트 그대로 | 거트명 |
| tension | String | orders.tension | 텍스트 (예: "25lbs", "25/27lbs") | 텐션값 |
| status | Enum | orders.status | 상태 뱃지 (Large) | `received` / `in_progress` / `completed` |
| memo | String (nullable) | orders.memo | 텍스트, 줄바꿈 유지 | 작업 메모. null이면 섹션 숨김 |
| created_at | Timestamp | orders.created_at | "MM/DD HH:mm" | 접수 시각 (타임라인 접수됨 노드) |
| updated_at | Timestamp | orders.updated_at | "MM/DD HH:mm" | 최종 상태 변경 시각 |
| received_at | Timestamp | orders.created_at | "MM/DD HH:mm" | 접수됨 도달 시각 (= created_at) |
| in_progress_at | Timestamp | orders.in_progress_at (nullable) | "MM/DD HH:mm" 또는 "—" | 작업중 도달 시각. 미도달 시 "—" |
| completed_at | Timestamp | orders.completed_at (nullable) | "MM/DD HH:mm" 또는 "—" | 완료 도달 시각. 미도달 시 "—" |
| shop_name | String | shops.name | 텍스트 | 샵 이름 |
| shop_address | String | shops.address | 텍스트 | 샵 주소 |
| shop_phone | String | shops.phone | 텍스트, 탭 시 전화 연결 | 샵 연락처 |
| shop_latitude | Double | shops.latitude | 내부 사용 (길찾기용) | 샵 위도 |
| shop_longitude | Double | shops.longitude | 내부 사용 (길찾기용) | 샵 경도 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 작업 상세 조회 | `orders` JOIN `shops` | `orders.id = order_id` | orders.*, shops.name, shops.address, shops.phone, shops.latitude, shops.longitude | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
final order = await supabase
  .from('orders')
  .select('''
    *,
    shops!inner(name, address, phone, latitude, longitude)
  ''')
  .eq('id', orderId)
  .single();
```

### 6.2 데이터 변경

이 화면에서 데이터 변경은 발생하지 않는다. (조회 전용 화면)

### 6.3 실시간 구독

| 구독 대상 | 테이블 | 조건 | 이벤트 | 처리 |
|-----------|--------|------|--------|------|
| 작업 상태 변경 | `orders` | `orders.id = order_id` | UPDATE (status 변경) | 상태 뱃지, 타임라인 즉시 갱신. 상태 전환 애니메이션 300ms |

**Supabase Realtime 구독 예시:**

```dart
supabase
  .from('orders')
  .stream(primaryKey: ['id'])
  .eq('id', orderId)
  .listen((List<Map<String, dynamic>> data) {
    if (data.isNotEmpty) {
      _updateOrderDetail(data.first);
    }
  });
```

## 9. 비고

### 길찾기 연동
- "길찾기" 버튼 탭 시 네이버 지도 앱의 길찾기를 호출한다 (`nmap://route/...`).
- 출발지는 현재 위치(GPS), 목적지는 샵의 위도/경도를 사용한다.
- 네이버 지도 앱 미설치 시 웹 브라우저로 네이버 지도 웹 버전을 폴백 호출한다.

### 타임라인 시각 정보
- `received_at`은 `orders.created_at`과 동일하다.
- `in_progress_at`, `completed_at`은 별도 컬럼이 없는 경우, 상태 변경 이력 로그 또는 `updated_at`으로 추정할 수 있다. 추후 상태 변경 이력 테이블 도입 시 정확한 시각 표시가 가능하다.

### 실시간 구독 정책
- 실시간 구독은 앱이 **포그라운드** 상태일 때만 활성화한다.
- 화면 진입 시 구독 시작, 화면 이탈 시 구독 해제한다.

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 타임라인의 색상 구분은 아이콘(체크/빈 원)으로도 구분 가능하다.
- 전화번호는 시맨틱 라벨("전화 연결")을 포함한다.

### 디자인 레퍼런스
- **토스**: 카드 기반의 정보 그룹핑, 깔끔한 섹션 구분
- **배민**: 둥근 모서리(16px), 친근한 UI
- **카카오택시**: 상태 타임라인 UX, 실시간 진행 현황 추적

### 성공/실패 피드백
- 에러 재시도 시 로딩 상태 전환 후 API 재호출한다. 성공 시 정상 화면으로 복귀하고, 실패 시 에러 상태를 다시 표시한다.
- 길찾기 버튼: 네이버 지도 앱 미설치 시 웹 브라우저로 폴백. 폴백 실패 시 "지도 앱을 열 수 없습니다" 스낵바 표시.
- 전화 연결: 전화 앱 실행 실패 시 "전화 앱을 열 수 없습니다" 스낵바 표시.

### Flutter 구현 참고
- 이 화면은 실시간 구독(stream)을 사용하므로 `StatefulWidget`으로 구현한다.
- 화면 진입 시 구독 시작, `dispose`에서 구독 해제한다.
