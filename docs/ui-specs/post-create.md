# 게시글 작성/수정 — UI 화면 스펙

> 최종 수정일: 2026-03-12

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-post-create` |
| **화면 명** | 게시글 작성/수정 |
| **목적** | 사장님이 공지사항 또는 이벤트 게시글을 작성하거나 기존 게시글을 수정한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`, 샵 등록 완료 상태 |
| **모드** | `postId` 파라미터 없음 = 작성 모드, `postId` 있음 = 수정 모드 |

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 버튼 + "게시글 작성" 타이틀 | 배경 `#FFFFFF15` |
| 카테고리 선택 | 공지사항 / 이벤트 선택 | 토글 형태, 반드시 하나 선택 |
| 제목 입력 | 게시글 제목 (필수) | 텍스트 필드 |
| 내용 입력 | 게시글 본문 (필수) | 멀티라인 텍스트 |
| 이미지 첨부 | 이미지 첨부 (선택, 최대 5장) | 가로 스크롤 이미지 리스트 |
| 이벤트 기간 | 이벤트 시작일~종료일 (이벤트 카테고리 선택 시만 표시) | 날짜 선택기 |
| 등록 버튼 | 게시글 등록 CTA 버튼 | 필수 필드 완료 시 활성화 |

| 9 | 뒤로가기 | 앱바 뒤로가기 또는 시스템 백 | 입력 내용 있으면 확인 다이얼로그 | "작성 중인 내용이 있습니다. 나가시겠습니까?" |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 게시글 조회 | `posts` | SELECT | `post_id` | 수정 모드 진입 시 (postId가 있는 경우) |

작성 모드에서는 초기 데이터 조회가 발생하지 않는다.

### 6.2 데이터 변경

| API | 테이블/서비스 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 이미지 업로드 | Supabase Storage (`post-images` 버킷) | UPLOAD | 이미지 바이트 데이터 (`Uint8List`) + 확장자 | 이미지 추가 시 즉시 업로드 (개별 업로드, 최대 5회) |
| 게시글 등록 | `posts` | INSERT | `{shop_id, category, title, content, images(URL 목록), event_start_date, event_end_date}` | 등록 버튼 탭 시 |

**Supabase 쿼리 예시:**

```dart
// 1. 이미지 업로드 (이미지 추가 시 즉시 수행 — addImage 메서드)
final userId = supabase.auth.currentUser!.id;
final ts = DateTime.now().microsecondsSinceEpoch;
final path = '$userId/$ts.$extension';
final url = await storageRepository.uploadImage('post-images', bytes, path);
// URL을 state.images에 추가

// 2. 게시글 등록 (submit 시 이미 업로드된 URL 목록 사용)
await postRepository.create(Post(
  shopId: shopId,
  category: state.category,
  title: state.title,
  content: state.content,
  images: state.images,  // 이미 업로드된 URL 목록
  eventStartDate: state.eventStartDate,
  eventEndDate: state.eventEndDate,
));
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **초기** | 화면 진입 직후 | 카테고리 미선택, 빈 폼, 등록 버튼 비활성 |
| **입력 중** | 필드 입력 진행 중 | 실시간 검증, 필수 필드 완료 시 등록 버튼 활성화 |
| **제출 중** | 등록 버튼 탭 후 API 호출 중 | 등록 버튼에 로딩 인디케이터, 입력 비활성화 |
| **에러** | API 호출 실패 | 에러 스낵바 표시, 등록 버튼 재활성화 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `category` | String? | null | 선택된 카테고리 (`notice` / `event`) |
| `title` | String | "" | 제목 입력값 |
| `content` | String | "" | 내용 입력값 |
| `images` | List\<String\> | [] | 첨부된 이미지 URL 목록 (업로드 즉시 Storage에 업로드하여 URL 저장) |
| `isUploadingImage` | bool | false | 이미지 업로드 중 여부 |
| `eventStartDate` | DateTime? | null | 이벤트 시작일 |
| `eventEndDate` | DateTime? | null | 이벤트 종료일 |
| `isSubmitting` | bool | false | 제출 중 여부 |
| `categoryError` | String? | null | 카테고리 미선택 에러 |
| `titleError` | String? | null | 제목 에러 메시지 |
| `contentError` | String? | null | 내용 에러 메시지 |
| `dateError` | String? | null | 날짜 에러 메시지 |

## 9. 비고

- **카테고리 선택 필수**: 카테고리(공지사항/이벤트)는 반드시 하나를 선택해야 게시글을 등록할 수 있다.
- **이벤트 기간**: 이벤트 카테고리 선택 시에만 기간 입력 영역이 표시된다. 기간은 시작일과 종료일로 구성되며, 종료일은 시작일 이후여야 한다.
- **이미지 첨부**: 최대 5장까지 첨부 가능하며, 각 이미지는 10MB 이하여야 한다. 카메라 촬영 또는 갤러리에서 선택 가능하다.
- **이미지 압축**: 업로드 전 이미지를 적절한 크기로 압축하여 저장 용량을 절약한다.
- **키보드 처리**: `SingleChildScrollView`로 감싸서 키보드가 올라올 때 스크롤 가능하게 한다.
- **작성 중 이탈 방지**: 입력 내용이 있는 상태에서 뒤로가기 시 확인 다이얼로그를 표시한다.

### 접근성
- 뒤로가기 아이콘 버튼에 접근성 라벨(`semanticLabel: "뒤로가기"`)을 포함한다.
- 이미지 추가 버튼에 접근성 라벨(`semanticLabel: "이미지 추가"`)을 포함한다.
- 이미지 삭제 버튼에 접근성 라벨(`semanticLabel: "이미지 삭제"`)을 포함한다.
- 카테고리 칩의 선택 상태는 색상 + 테두리 두께 변화로 구분하여, 색상만으로 정보를 전달하지 않는다.

### 로딩 및 피드백
- 등록 버튼 탭 시 버튼 내 `CircularProgressIndicator`를 표시하고, 중복 탭을 방지한다 (`isSubmitting` 플래그 활용).
- 이미지 업로드 중 진행 상태를 사용자에게 표시한다 (버튼 로딩 인디케이터로 갈음).
- 등록 성공 시 "게시글이 등록되었습니다" 토스트, 실패 시 에러 스낵바를 표시한다.
- 입력 검증 에러는 해당 필드 바로 아래에 에러 메시지를 표시한다 (`titleError`, `contentError`, `dateError` 등 활용).

### Flutter 구현 참고
- 이 화면은 폼 입력이 있으므로 `StatefulWidget`을 사용하고, `Form` + `GlobalKey<FormState>`로 유효성 검증을 수행한다.
- 이미지 리스트는 가로 스크롤이므로 `ListView.builder(scrollDirection: Axis.horizontal)`을 사용한다.
