# 샵 상세 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-shop-detail` |
| **화면 명** | 샵 상세 |
| **목적** | 선택한 샵의 상세 정보(소개, 위치, 연락처)와 현재 작업 현황을 확인하고, 길찾기를 통해 샵을 방문할 수 있도록 한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능. `shop_id` 필수 전달 |

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| shop_name | String | shops.name | 텍스트 그대로 | 샵 이름 |
| shop_description | String (nullable) | shops.description | 텍스트, 줄바꿈 유지 | 샵 소개글. null이면 숨김 |
| shop_address | String | shops.address | 텍스트 그대로 | 샵 주소 (전체) |
| shop_phone | String | shops.phone | 텍스트, 탭 시 전화 연결 | 샵 연락처 |
| shop_latitude | Double | shops.latitude | 내부 사용 (지도, 길찾기) | 샵 위도 |
| shop_longitude | Double | shops.longitude | 내부 사용 (지도, 길찾기) | 샵 경도 |
| received_count | Int | orders 집계 | "N" (숫자만 크게, 하단 "접수" 라벨) | 접수됨 상태 작업 건수 |
| in_progress_count | Int | orders 집계 | "N" (숫자만 크게, 하단 "작업중" 라벨) | 작업중 상태 작업 건수 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 샵 상세 조회 | `shops` | `shops.id = shop_id` | shops.* | 화면 진입 시 |
| 샵 작업 현황 조회 | `orders` | `orders.shop_id = shop_id` AND `status IN ('received', 'in_progress')` | COUNT by status | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
// 샵 상세 조회
final shop = await supabase
  .from('shops')
  .select('*')
  .eq('id', shopId)
  .single();

// 작업 현황 조회
final orders = await supabase
  .from('orders')
  .select('status')
  .eq('shop_id', shopId)
  .in_('status', ['received', 'in_progress']);

final receivedCount = orders.where((o) => o['status'] == 'received').length;
final inProgressCount = orders.where((o) => o['status'] == 'in_progress').length;
```

이 화면에서 데이터 변경은 발생하지 않는다. (조회 전용 화면)

## 9. 비고

### 길찾기 연동
- "길찾기" 버튼 탭 시 네이버 지도 앱의 길찾기를 호출한다 (`nmap://route/...`).
- 출발지는 현재 위치(GPS), 목적지는 샵의 위도/경도를 사용한다.
- 네이버 지도 앱 미설치 시 웹 브라우저로 네이버 지도 웹 버전을 폴백 호출한다.

### 지도 미리보기
- NaverMap 위젯을 정적(static) 모드로 사용한다.
- 스크롤, 줌, 회전 등 모든 제스처를 비활성화하여 상위 스크롤과 충돌하지 않도록 한다.
- 지도 탭 시 네이버 지도 앱으로 이동하여 해당 위치를 표시한다.

### 작업 현황 표시 정책
- 접수 건수와 작업중 건수는 화면 진입 시 1회 조회한다.
- 실시간 구독은 하지 않는다 (샵 상세 화면은 일시적 조회 화면).

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 전화번호는 시맨틱 라벨("전화 연결")을 포함한다.
- 지도 미리보기 영역에 접근성 라벨("샵 위치 지도, 탭하여 네이버 지도에서 열기")을 포함한다.

### 디자인 레퍼런스
