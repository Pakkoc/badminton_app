# 샵 상세 — UI 화면 스펙

> 최종 수정일: 2026-03-03

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-shop-detail` |
| **화면 명** | 샵 상세 |
| **목적** | 선택한 샵의 상세 정보(소개, 위치, 연락처)와 현재 작업 현황을 확인하고, 공지사항/이벤트/가게 재고를 카테고리별로 탐색하며, 길찾기를 통해 샵을 방문할 수 있도록 한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능. `shop_id` 필수 전달 |

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 버튼 + "샵 정보" 타이틀 | 앱바 배경 `#FFFFFF`, 하단 테두리 `#E8E0D8` 0.5px |

| 위치 및 연락처 | 주소, 전화번호 표시 | 전화번호 탭 시 전화 앱 실행 |

### 3.11 로딩 상태 (Skeleton)

최초 진입 시 스켈레톤 shimmer를 표시한다.

| 컴포넌트 | 설명 |
|----------|------|
| 지도 스켈레톤 | 높이 180px, 너비 100%, 배경 `#E8E0D8` |
| 샵 이름 스켈레톤 | 높이 32px, 너비 60%, 모서리 8px |
| 소개글 스켈레톤 | 높이 40px, 너비 100%, 모서리 8px |
| 작업 현황 스켈레톤 | 높이 80px, 너비 100%, 모서리 20px |
| 위치/연락처 스켈레톤 | 높이 80px, 너비 100%, 모서리 20px |

**스켈레톤 스타일:**

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 샵 상세 조회 | `shops` | `shops.id = shop_id` | shops.* | 화면 진입 시 |
| 샵 작업 현황 조회 | `orders` | `orders.shop_id = shop_id` AND `status IN ('received', 'in_progress')` | COUNT by status | 화면 진입 시 |
| 공지사항 목록 조회 | `posts` | `posts.shop_id = shop_id` AND `category = 'notice'` ORDER BY `created_at` DESC | posts.* | 공지사항 탭 선택 시 (또는 화면 진입 시 미리 로드) |
| 이벤트 목록 조회 | `posts` | `posts.shop_id = shop_id` AND `category = 'event'` ORDER BY `created_at` DESC | posts.* | 이벤트 탭 선택 시 (또는 화면 진입 시 미리 로드) |
| 재고 목록 조회 | `inventory` | `inventory.shop_id = shop_id` ORDER BY `name` ASC | inventory.* | 가게 재고 탭 선택 시 (또는 화면 진입 시 미리 로드) |

**Supabase 쿼리 예시:**

```dart
// 샵 상세 조회
final shop = await supabase
  .from('shops')
  .select('*')
  .eq('id', shopId)
  .single();

// 작업 현황 조회
final orders = await supabase
  .from('orders')
  .select('status')
  .eq('shop_id', shopId)
  .in_('status', ['received', 'in_progress']);

final receivedCount = orders.where((o) => o['status'] == 'received').length;
final inProgressCount = orders.where((o) => o['status'] == 'in_progress').length;

// 공지사항 목록 조회
final notices = await supabase
  .from('posts')
  .select('*')
  .eq('shop_id', shopId)
  .eq('category', 'notice')
  .order('created_at', ascending: false);

// 이벤트 목록 조회
final events = await supabase
  .from('posts')
  .select('*')
  .eq('shop_id', shopId)
  .eq('category', 'event')
  .order('created_at', ascending: false);

// 재고 목록 조회
final inventory = await supabase
  .from('inventory')
  .select('*')
  .eq('shop_id', shopId)
  .order('name', ascending: true);
```

## 9. 비고

### 길찾기 연동
- "길찾기" 버튼 탭 시 네이버 지도 앱의 길찾기를 호출한다 (`nmap://route/...`).
- 출발지는 현재 위치(GPS), 목적지는 샵의 위도/경도를 사용한다.
- 네이버 지도 앱 미설치 시 웹 브라우저로 네이버 지도 웹 버전을 폴백 호출한다.

### 지도 미리보기
- NaverMap 위젯을 정적(static) 모드로 사용한다.
- 스크롤, 줌, 회전 등 모든 제스처를 비활성화하여 상위 스크롤과 충돌하지 않도록 한다.
- 지도 탭 시 네이버 지도 앱으로 이동하여 해당 위치를 표시한다.

### 카테고리 탭
- 기본 선택 탭은 "공지사항"이다.
- 탭 전환 시 스와이프 제스처를 지원한다 (TabBarView).
- 각 탭의 데이터는 화면 진입 시 한 번에 로드하거나, 탭 선택 시 지연 로드(lazy load) 할 수 있다.

### 가게 재고 열람 정책
- 고객은 재고 정보를 열람만 할 수 있으며 구매 기능은 제공하지 않는다.
- 재고 이미지가 없는 상품은 기본 이미지(아이콘)를 표시한다.

### 작업 현황 표시 정책
- 접수 건수와 작업중 건수는 화면 진입 시 1회 조회한다.
- 실시간 구독은 하지 않는다 (샵 상세 화면은 일시적 조회 화면).

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 전화번호는 시맨틱 라벨("전화 연결")을 포함한다.
- 지도 미리보기 영역에 접근성 라벨("샵 위치 지도, 탭하여 네이버 지도에서 열기")을 포함한다.

### 디자인 레퍼런스
- **토스**: 카드 기반의 정보 그룹핑, 깔끔한 섹션 구분
- **배민**: 둥근 모서리(16px), 매장 상세 페이지 레이아웃, 탭 기반 카테고리 전환
- **카카오맵**: 장소 상세의 지도 미리보기 + 정보 조합 패턴

### 성공/실패 피드백
- 에러 재시도 시 로딩 상태 전환 후 API 재호출. 성공 시 정상 화면, 실패 시 에러 상태 재표시.
- 길찾기 버튼: 네이버 지도 앱 미설치 시 웹 브라우저 폴백. 폴백 실패 시 "지도 앱을 열 수 없습니다" 스낵바 표시.
- 전화 연결: 전화 앱 실행 실패 시 "전화 앱을 열 수 없습니다" 스낵바 표시.

### Flutter 구현 참고
- 이 화면은 탭 전환(TabController) + 여러 상태를 관리하므로 `StatefulWidget`으로 구현한다.
- 공지사항/이벤트 리스트는 `ListView.builder`, 재고 그리드는 `GridView.builder`로 구현한다.
