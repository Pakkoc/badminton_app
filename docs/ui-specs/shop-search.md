# 주변 샵 검색 — UI 화면 스펙

> 최종 수정일: 2026-02-22

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-shop-search` |
| **화면 명** | 주변 샵 검색 |
| **목적** | 현재 위치 기반으로 주변 거트 샵을 지도 또는 리스트에서 검색하고, 각 샵의 작업 현황을 확인한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능 |

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| shop_name | String | shops.name | 텍스트 그대로 | 샵 이름 |
| shop_address | String | shops.address | 주소 앞부분 (구/동까지) | 샵 주소. 지면 절약을 위해 간략 표시 |
| shop_latitude | Double | shops.latitude | 내부 사용 (마커 위치, 거리 계산) | 샵 위도 |
| shop_longitude | Double | shops.longitude | 내부 사용 (마커 위치, 거리 계산) | 샵 경도 |
| distance | Double | 클라이언트 계산 | 거리 표시 규칙 참조 | 현재 위치로부터 샵까지 직선 거리 |
| received_count | Int | orders 집계 | "접수 N건" | 해당 샵의 접수됨 상태 작업 건수 |
| in_progress_count | Int | orders 집계 | "작업중 N건" | 해당 샵의 작업중 상태 작업 건수 |

| 6 | 지도 이동/줌 | 지도 드래그 또는 핀치 줌 | 보이는 영역 내 샵 재검색 | 지도 이동 완료 후 500ms debounce 후 화면 영역 내 샵 조회 |
| 7 | 하단 탭 전환 | 하단 네비게이션 탭 터치 | 해당 화면으로 이동 | 선택된 탭 화면으로 전환 |
| 8 | 위치 권한 요청 | 화면 최초 진입 시 | 위치 권한 팝업 표시 | 허용 시 지도 표시, 거부 시 위치 권한 미허용 상태 표시 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 주변 샵 목록 조회 | `shops` | 위도/경도 범위 필터 (지도 화면 영역 기준) | shops.*, 각 샵별 접수/작업중 건수 | 화면 진입 시, 지도 이동 시 (debounce 500ms) |
| 샵별 작업 현황 조회 | `orders` | `orders.shop_id = shop.id` AND `status IN ('received', 'in_progress')` | COUNT by status, grouped by shop_id | 샵 목록 조회와 함께 |

**Supabase 쿼리 예시:**

```dart
// 주변 샵 목록 조회 (위도/경도 범위)
final shops = await supabase
  .from('shops')
  .select('*')
  .gte('latitude', southLat)
  .lte('latitude', northLat)
  .gte('longitude', westLng)
  .lte('longitude', eastLng);

// 샵별 작업 현황은 RPC 함수 또는 클라이언트에서 개별 조회
final shopIds = shops.map((s) => s['id']).toList();
final orderCounts = await supabase
  .rpc('get_shop_order_counts', params: {'shop_ids': shopIds});
```

## 9. 비고

### 위치 권한 처리
- 화면 최초 진입 시 위치 권한을 요청한다.
- 권한 허용 시: 현재 위치를 중심으로 지도 표시 및 주변 샵 조회.
- 권한 거부 시: 위치 권한 안내 화면 표시. "설정으로 이동" 버튼으로 시스템 설정에서 권한 변경 유도.
- 앱 설정에서 권한 변경 후 복귀 시 자동으로 위치 재확인.

### 지도 영역 기반 검색
- 지도를 이동하거나 줌 변경 시, 화면에 보이는 영역(bounds) 내의 샵만 조회한다.
- 지도 이동 완료 후 500ms debounce를 적용하여 불필요한 API 호출을 방지한다.
- 리스트 뷰에서는 현재 위치 기준 반경 5km 이내의 샵을 거리순으로 정렬한다.

### 작업 현황 표시 정책
- 각 샵의 접수 건수와 작업중 건수는 실시간 데이터가 아닌 조회 시점의 스냅샷이다.
- 화면 진입 시 1회 조회하며, 실시간 구독은 하지 않는다.

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 마커는 색상 + 아이콘으로 구분 가능하다.
- 리스트 뷰는 스크린 리더 지원을 위해 카드 전체에 시맨틱 라벨을 포함한다.

### 디자인 레퍼런스
- **카카오맵**: 지도 + 하단 시트 조합의 검색 UX
- **배민**: 둥근 카드(16px), 간결한 정보 표시
- **당근마켓**: 현재 위치 기반 주변 검색 패턴

### 성공/실패 피드백
- 지도 로드 실패 시 "지도를 불러올 수 없습니다" 메시지 + 재시도 버튼 표시.
- 샵 목록 조회 실패 시 "데이터를 불러올 수 없습니다" 스낵바 표시 후, 기존 마커/리스트 유지.
- 현재 위치 버튼 탭 시 GPS 위치 획득 실패 시 "현재 위치를 확인할 수 없습니다" 스낵바 표시.

### Flutter 구현 참고
- 이 화면은 지도 상태 + 뷰 모드 전환 + 위치 권한 등 복잡한 상태를 관리하므로 `StatefulWidget`으로 구현한다.
- 리스트 뷰의 샵 목록은 `ListView.builder`로 구현하여 효율적으로 렌더링한다.
