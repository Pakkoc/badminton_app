# 알림 — UI 화면 스펙

> 최종 수정일: 2026-03-03

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-notifications` |
| **화면 명** | 알림 |
| **목적** | 주문 상태 변경, 새 게시글 등의 알림을 시간순으로 확인한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능 |

│                             │
│  알림 리스트 (스크롤 영역)      │

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 + "알림" 타이틀 | 배경 `#FFFFFF`, 하단 테두리 `#E8E0D8` 0.5px |
| 알림 리스트 | 알림 아이템을 시간순(최신 먼저)으로 표시 | divider로 구분, 하단 네비게이션 없음 (상세 화면) |

## 3. 컴포넌트 목록

### 3.1 상단 앱바

| 컴포넌트 | 타입 | 설명 | 스타일 |
|----------|------|------|--------|
| 뒤로가기 | IconButton | 이전 화면으로 복귀 | Material Icon `arrow_back_ios`, 크기 24px, 색상 `#1A1A2E` |
| 타이틀 | Text | "알림" | `titleLarge` (20sp, Bold), 색상 `#1A1A2E` |

**앱바 스타일:**

| 속성 | 값 |
|------|-----|
| 배경색 | `#FFFFFF` |
| 높이 | 56px |
| 패딩 | [0, 16] |

### 3.2 알림 아이템

| 컴포넌트 | 타입 | 설명 | 스타일 |
|----------|------|------|--------|
| 아이콘 원형 배경 | Container | 알림 유형별 아이콘 | 40x40px, 원형 (cornerRadius 20px), 유형별 배경색 |
| 알림 제목 | Text | 알림 제목 | `bodyMedium` (14sp, Medium 500), 색상 `#1A1A2E` |
| 알림 내용 | Text | 알림 상세 내용 | `bodySmall` (13sp), 색상 `#4A4A5A`, 최대 2줄 |
| 시간 | Text | 상대 시간 표시 | `bodySmall` (12sp), 색상 `#9CA3AF` |
| 하단 구분선 | Divider | 아이템 간 구분 | 색상 `#EFF6FF`, 두께 1px |

**알림 아이템 레이아웃:**
- Row: [아이콘 원형 40x40] + [Column: 제목, 내용, 시간]
- 아이콘과 텍스트 간격: 12px
- 아이템 내부 패딩: 16px
- 제목 ~ 내용: 4px
- 내용 ~ 시간: 4px

### 3.3 알림 유형별 아이콘

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| type | Enum | notifications.type | 아이콘 유형 결정 | 알림 유형 |
| title | String | notifications.title | 텍스트 | 알림 제목 |
| body | String | notifications.body | 텍스트, 최대 2줄 | 알림 내용 |
| created_at | Timestamp | notifications.created_at | 상대시간 | 알림 시간 |
| order_id | String? | notifications.order_id | 내부 사용 (네비게이션) | 연결된 주문 ID |
| is_read | bool | notifications.is_read | 읽지 않은 알림 강조 | 읽음 여부 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 알림 목록 조회 | `notifications` | `user_id = 현재_사용자_id` ORDER BY `created_at` DESC | notifications.* | 화면 진입 시 |

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 알림 읽음 처리 | `notifications` | UPDATE | `{is_read: true}` | 알림 아이템 탭 시 |
| 전체 읽음 처리 | `notifications` | UPDATE | `{is_read: true}` (전체) | 화면 진입 시 |

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 데이터 로드 중 | 스켈레톤 shimmer |

| **에러** | 네트워크 오류 | 에러 메시지 + 재시도 버튼 |

## 9. 비고

- 알림 화면 진입 시 모든 알림을 **자동 읽음 처리**한다. 고객 홈의 알림 뱃지 숫자가 0으로 초기화된다.
- 하단 네비게이션 바는 표시하지 않는다 (상세 화면).
- 알림 목록은 **최신순 정렬**(created_at DESC)이다.

### 접근성
- 뒤로가기 버튼에 `semanticLabel: "뒤로가기"`를 포함한다.
- 알림 아이콘에 알림 유형을 설명하는 접근성 라벨을 포함한다.

### Flutter 구현 참고
- `ListView.builder`로 알림 목록을 구현한다.
- `StatefulWidget`으로 구현 (로딩/빈/에러 상태 관리).
