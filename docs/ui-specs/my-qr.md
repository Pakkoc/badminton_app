# 내 QR코드 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-my-qr` |
| **화면 명** | 내 QR코드 |
| **목적** | 고객이 샵 사장님에게 보여줄 QR코드를 표시하여, QR 스캔을 통한 간편 회원 등록 및 작업 접수를 지원한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능 |

## 4. 필드 정의

이 화면은 입력 필드가 없는 표시 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| user_id | UUID | Supabase Auth (현재 로그인 사용자) | QR코드로 인코딩 | QR코드에 담기는 유일한 데이터. 사장님이 스캔하여 회원 조회/등록에 사용 |
| user_name | String | users.name | 텍스트 그대로 | QR코드 아래에 표시되는 사용자 이름 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 현재 사용자 정보 | Supabase Auth + `users` | 현재 로그인 사용자 | user_id, name | 화면 진입 시 (캐시 사용 가능) |

**데이터 소스:**

```dart
// user_id는 Supabase Auth에서 직접 취득
final userId = supabase.auth.currentUser!.id;

// 사용자 이름은 users 테이블 또는 로컬 캐시에서 취득
final userName = userProvider.currentUser.name;
```

이 화면에서 데이터 변경은 발생하지 않는다. (표시 전용 화면)

## 9. 비고

### QR코드 보안 정책
- QR코드에는 `user_id` (UUID)만 인코딩한다. 이름, 연락처 등 개인정보는 포함하지 않는다.
- 사장님이 QR을 스캔하면 `user_id`로 Supabase에서 사용자 정보를 조회하여 회원 등록에 활용한다.
- QR코드는 고정값이므로 매번 새로 생성할 필요 없이 캐시하여 사용한다.

### 화면 밝기 자동 조절
- 화면 진입 시 시스템 밝기를 1.0 (최대)으로 변경한다.
- QR코드 인식률을 높이기 위한 조치이다.
- 화면 이탈 시 (다른 탭 이동, 앱 백그라운드 전환 등) 반드시 이전 밝기 값으로 복원한다.
- Flutter `screen_brightness` 패키지 또는 플랫폼 채널을 사용한다.

### 화면 잠금 방지
- QR코드 화면이 활성화된 동안에는 화면 자동 잠금(Screen Timeout)을 방지한다.
- `WakelockPlus` 패키지 등을 사용하여 화면이 꺼지지 않도록 한다.
- 화면 이탈 시 Wakelock을 해제한다.

### 접근성
- QR코드 이미지에 접근성 라벨("내 QR코드, 사장님에게 보여주세요")을 포함한다.
- 화면 밝기 변경은 시스템 접근성 설정에 영향을 주지 않는다.
