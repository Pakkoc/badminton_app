|------|------|
| **화면 ID** | `owner-shop-qr` |
| **화면 명** | 샵 QR코드 |

## 9. 비고

### QR코드 동작 정책
- QR코드에는 샵 고유 딥링크 URL(`https://gutalarm.app/shop/{shop_id}`)을 인코딩한다.
- **앱 설치 고객이 스캔**: 앱이 열리며 QR 접수 화면(`customer-qr-order`)으로 이동한다. 자동으로 회원 등록(미등록 시) + 작업 접수(received 상태)가 진행되며, 완료 후 "접수 완료" 안내 다이얼로그를 표시하고 고객 홈으로 이동한다.
- **앱 미설치 고객이 스캔**: 앱 다운로드 페이지(앱스토어/플레이스토어)로 이동한다. 앱 설치 후 최초 실행 시 딥링크가 처리되어 해당 샵에 자동 등록된다.

### 인쇄용 QR코드
- 인쇄용 다운로드 시 A4 용지에 인쇄했을 때 적절한 크기가 되도록 고해상도(최소 1024x1024px) PNG 이미지를 생성한다.
- QR코드 하단에 샵 이름과 "거트알림 앱으로 스캔하세요" 안내 문구를 포함한다.

### 공유 기능
- 시스템 공유 시트를 사용하여 카카오톡, 문자, 이메일 등 다양한 채널로 QR코드 이미지를 공유할 수 있다.
- 공유 시 QR코드 이미지와 함께 "거트알림 앱으로 QR을 스캔하면 회원 등록이 됩니다" 텍스트를 첨부한다.

### 접근성
- QR코드 이미지에 접근성 라벨("샵 QR코드, 고객이 스캔하여 회원 등록")을 포함한다.
- 버튼에 접근성 라벨("QR코드 공유", "인쇄용 QR코드 다운로드")을 포함한다.
- 뒤로가기 아이콘 버튼에 접근성 라벨(`semanticLabel: "뒤로가기"`)을 포함한다.
- 하단 네비게이션 아이콘에 접근성 라벨을 포함한다 (각 탭명과 동일).
- QR 동작 설명의 정보는 아이콘과 텍스트를 병행하여 색상만으로 구분하지 않는다.

### 로딩 및 피드백
- 화면 진입 시 로컬 캐시에서 데이터를 취득하므로 별도 로딩 상태가 불필요하나, 캐시 미스 시 `CircularProgressIndicator`를 중앙에 표시한다.
- 인쇄용 다운로드 시 버튼에 로딩 상태를 표시하고, 중복 탭을 방지한다 (`isSaving` 플래그 활용).
- 공유 버튼도 공유 시트 호출 중 중복 탭을 방지한다.

### Flutter 구현 참고
- 이 화면은 사용자 입력이 없으므로 `StatelessWidget` + Provider/Riverpod 조합을 권장한다.
- QR코드 렌더링에는 `qr_flutter` 패키지를 사용한다.
