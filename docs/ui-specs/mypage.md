# 마이페이지 — UI 화면 스펙

> 최종 수정일: 2026-03-12

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-mypage` |
| **화면 명** | 마이페이지 |
| **목적** | 사용자의 프로필 정보를 확인/수정하고, 알림 설정, 샵 사장님 등록/모드 전환 및 앱 정보를 관리할 수 있도록 한다 |
| **사용자 역할** | 공통 (customer / shop_owner) |
| **진입 조건** | 로그인 필수 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 사용자 프로필 조회 | `users` | `id = 내_user_id` | name, phone | 화면 진입 시 |
| 샵 등록 상태 조회 | `shops` | `owner_id = 내_user_id` | status, reject_reason | 화면 진입 시 (사장님 역할) |

**Supabase 쿼리 예시:**

```dart
// 사용자 프로필 조회
final profile = await supabase
  .from('users')
  .select('name, phone')
  .eq('id', myUserId)
  .single();

// 이메일은 Supabase Auth에서 조회
final email = supabase.auth.currentUser?.email;
```

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 프로필 수정 | `users` | UPDATE | `{name: 이름, phone: 연락처}` | 프로필 수정 저장 시 |
| FCM 토큰 등록/삭제 | `users` | UPDATE | `{fcm_token: 토큰 or null}` | 푸시 알림 토글 변경 시 |

**Supabase 쿼리 예시:**

```dart
// 프로필 수정
await supabase
  .from('users')
  .update({'name': newName, 'phone': newPhone})
  .eq('id', myUserId);

// FCM 토큰 등록
await supabase
  .from('users')
  .update({'fcm_token': token})
  .eq('id', myUserId);
```

### 6.3 실시간 구독

이 화면에서는 실시간 구독이 없다.

## 9. 비고

- 프로필 수정은 **별도 화면(`customer-profile-edit`)**으로 이동하여 수정한다. 프로필 이미지 변경, 이름/연락처 수정 기능을 제공한다.
- 로그아웃 시 **확인 다이얼로그**를 반드시 표시하여 오조작을 방지한다. "로그아웃 하시겠습니까?" + "취소" / "확인" 버튼 구성.
- 푸시 알림 토글은 **즉시 반영**한다. on으로 변경 시 FCM 토큰을 서버에 등록하고, off로 변경 시 서버에서 토큰을 삭제한다.
- 앱 버전은 Flutter의 `package_info_plus` 패키지를 통해 동적으로 가져온다.
- 프로필 수정 시 연락처 변경은 기존 `members` 테이블과의 매칭에 영향을 줄 수 있으므로, 연락처 변경 시 관련 member 레코드도 함께 업데이트해야 한다.
- 디자인 참고: 토스의 마이페이지, 간결하고 그룹화된 설정 리스트 스타일.

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 프로필 아이콘(`person`)에 접근성 라벨 "프로필 이미지"를 포함한다.
- 푸시 알림 토글 스위치에 접근성 라벨 "푸시 알림 켜기/끄기"를 포함한다.
- 로그아웃 버튼의 빨간색(`#EF4444`) 텍스트는 색상 + "로그아웃" 텍스트로 구분하여, 색상만으로 정보를 전달하지 않는다.

### 성공/실패 피드백
- 프로필 수정 성공: "프로필이 수정되었습니다" 스낵바 (배경 `#10B981`, 텍스트 `#FFFFFF`). 바텀시트 닫힘.
- 프로필 수정 실패: "프로필 수정에 실패했습니다. 다시 시도해 주세요." 스낵바 (배경 `#EF4444`, 텍스트 `#FFFFFF`). 바텀시트 유지.
- 로그아웃 실패: "로그아웃에 실패했습니다" 스낵바 표시.
- 푸시 알림 토글 변경 실패: 토글을 이전 상태로 되돌리고 "알림 설정 변경에 실패했습니다" 스낵바 표시.

### Flutter 구현 참고
- 이 화면은 프로필 수정 바텀시트, 토글 상태 등을 관리하므로 `StatefulWidget`으로 구현한다.
- 프로필 수정 바텀시트의 폼은 `Form` + `GlobalKey<FormState>`를 사용하여 유효성 검증한다.
