# 게시글 관리 — UI 화면 스펙

> 최종 수정일: 2026-03-12

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-post-manage` |
| **화면 명** | 게시글 관리 |
| **목적** | 사장님이 자기 샵의 게시글 목록을 조회, 카테고리 필터링, 수정/삭제를 수행한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`, 샵 등록 완료 상태 |

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 뒤로가기 버튼 + "게시글 관리" 타이틀 | 배경 `$--surface` |
| 카테고리 탭 | 전체 / 공지사항 / 이벤트 | 3등분 탭, 활성 탭 하단 `#F59E0B` 2px |

## 4. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 카테고리 필터 | 탭 선택 | `filterByCategory(shopId, category)` | 선택 카테고리로 목록 갱신 |
| 2 | 게시글 편집 | 편집(✏️) 아이콘 탭 | `push('/post-manage/edit/:postId')` | PostCreateScreen (수정 모드) 진입 |
| 3 | 게시글 삭제 | 삭제(🗑️) 아이콘 탭 | ConfirmDialog 표시 → 확인 시 `deletePost(shopId, postId)` | 삭제 성공 시 목록 갱신 + 토스트 |
| 4 | 새 게시글 작성 | FAB 탭 | `push('/post-manage/create')` | PostCreateScreen (작성 모드) 진입 |
| 5 | 작성/수정 복귀 | PostCreateScreen에서 `pop(true)` | `push<bool>` 결과 확인 → 목록 재로드 | 목록 갱신 |
| 6 | 뒤로가기 | 앱바 뒤로가기 | `pop()` | 샵 설정 화면으로 복귀 |

### 5.1 데이터 조회

## 6. 상태 관리

### 6.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 목록 조회 중 | LoadingIndicator |

| **에러** | API 호출 실패 | 에러 토스트 |

### 6.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `posts` | `List<Post>` | `[]` | 게시글 목록 |

## 8. 비고

- **삭제 확인**: 삭제 시 반드시 `ConfirmDialog`로 사용자 확인을 받는다.
- **목록 갱신**: 작성/수정/삭제 후 목록을 자동 갱신한다.
- **카테고리 탭**: "전체"는 `null`로 전달하여 필터 없이 전체 조회한다.
- **정렬**: `created_at` 기준 최신순 (내림차순).
