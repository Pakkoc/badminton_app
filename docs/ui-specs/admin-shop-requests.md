| **화면 ID** | `admin-shop-requests` |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 샵 등록 요청 목록 | `shops` | 필터 조건 (status) | id, name, phone, address, status, business_number, created_at | 화면 진입 시, 필터 변경 시, 새로고침 시 |
| 대기 건수 | `shops` | `status = 'pending'` | count | 화면 진입 시 (필터 탭 뱃지용) |

**Supabase 쿼리 예시:**

```dart
// 전체 목록 조회 (최신순)
final shops = await supabase
  .from('shops')
  .select('id, name, phone, address, status, business_number, created_at')
  .order('created_at', ascending: false);

// 상태별 필터링
final pendingShops = await supabase
  .from('shops')
  .select('id, name, phone, address, status, business_number, created_at')
  .eq('status', 'pending')
  .order('created_at', ascending: false);

// 대기 건수
final count = await supabase
  .from('shops')
  .select('id', const FetchOptions(count: CountOption.exact, head: true))
  .eq('status', 'pending');
```

## 9. 비고

- 관리자(admin) 전용 화면으로, role이 `admin`인 사용자만 접근 가능하다.
- 목록은 최신순(created_at DESC)으로 정렬한다.
- "대기" 필터 탭에는 대기 건수를 뱃지로 표시하여 빠르게 확인할 수 있도록 한다.
- 커서 기반 페이지네이션을 사용하여 대량의 요청도 성능 저하 없이 조회한다.
- 상세 화면에서 승인/거절 처리 후 돌아오면 목록을 자동 갱신한다.

### 접근성

- 상태 뱃지는 색상 + 텍스트("대기"/"승인"/"거절")로 구분하여 색상만으로 정보를 전달하지 않는다.
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 카드에 `semanticLabel: "샵 이름, 상태"` 를 적용한다.

### 로딩 상태 (스켈레톤)

- 카드 스켈레톤: shimmer 효과(배경 `#E8E0D8` -> `#EFF6FF`, 1500ms 반복), 3개 표시

### Flutter 구현 참고

- 목록은 `ListView.builder`로 구현하여 lazy loading을 적용한다.
- 필터 탭은 `FilterChip` 또는 커스텀 탭으로 구현한다.
- `RefreshIndicator`로 당겨서 새로고침을 지원한다.
