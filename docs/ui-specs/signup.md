# 프로필 설정 — UI 화면 스펙

> 최종 수정일: 2026-03-03

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `profile-setup` |
| **화면 명** | 프로필 설정 |
| **목적** | 소셜 로그인으로 인증된 신규 사용자가 역할(고객/샵 사장님)을 선택하고 이름, 연락처를 입력하여 프로필을 완성한다. 고객은 1단계에서 완료, 사장님은 2단계(샵 등록)로 이어진다 |
| **사용자 역할** | 공통 |
| **진입 조건** | 소셜 로그인 인증 완료 + 신규 사용자 (users 테이블에 레코드 없음) |

## 2. 레이아웃 구조

```
┌─────────────────────────────┐
│  ← 뒤로가기   "프로필 설정"   │  ← 앱바
├─────────────────────────────┤
│  ● 정보 입력 ─── ○ 샵 등록   │  ← 스텝 인디케이터 (사장님 선택 시만 표시)
├─────────────────────────────┤
│                             │
│  "서비스 이용을 위해          │
│   정보를 입력해주세요"        │  ← 서브타이틀
│                             │
│  "어떤 사용자이신가요?"       │
│                             │
│  ┌───────────┐ ┌──────────┐│
│  │  🏸        │ │  🏪      ││
│  │  고객      │ │ 샵 사장님 ││
│  │  거트 작업  │ │ 작업 접수 ││
│  │  현황 확인  │ │ 및 관리  ││
│  └───────────┘ └──────────┘│
│                             │
│  ┌───────────────────────┐  │
│  │ 이름 입력              │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 연락처 입력             │  │
│  └───────────────────────┘  │
│                             │
│  ┌───────────────────────┐  │
│  │  고객: "시작하기" 버튼  │  │
│  │  사장님: "다음" 버튼    │  │
│  └───────────────────────┘  │
│                             │
└─────────────────────────────┘
```

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 앱바 | 뒤로가기 아이콘 + "프로필 설정" 타이틀 | 뒤로가기 시 로그인(소셜 로그인) 화면으로 이동 |
| 스텝 인디케이터 | 사장님 역할 선택 시 2단계 진행 현황 표시 | 고객 선택 시 숨김 |
| 서브타이틀 | "서비스 이용을 위해 정보를 입력해주세요" | 항상 표시 |
| 역할 선택 영역 | 고객/샵 사장님 역할 선택 카드 2개 | 가로로 나란히 배치, 반드시 하나 선택 |
| 정보 입력 영역 | 이름, 연락처 입력 폼 | 역할 선택 후 표시 (또는 항상 표시) |
| 시작하기/다음 버튼 영역 | 역할에 따라 버튼 텍스트 변경 | 고객: "시작하기", 사장님: "다음" |

## 4. 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 검증 규칙 | 설명 |
|--------|------|------|--------|-----------|------|
| role | Enum | Y | null | `customer` 또는 `shop_owner` 중 하나 선택 | 사용자 역할 |
| name | String | Y | 소셜 프로필 이름 | 최소 2자, 최대 20자 | 사용자 이름 |
| phone | String | Y | "" | 한국 휴대폰 번호 형식 (010-XXXX-XXXX) | 연락처 |

### 검증 규칙 상세

| 필드 | 검증 시점 | 규칙 | 에러 메시지 |
|------|-----------|------|-------------|
| role | 시작하기/다음 버튼 탭 시 | 선택 여부 확인 | "사용자 유형을 선택해주세요" |
| name | 포커스 해제 시 + 버튼 탭 시 | 빈 값 | "이름을 입력해주세요" |
| name | 포커스 해제 시 | 2~20자 | "이름은 2자 이상 20자 이하로 입력해주세요" |
| phone | 포커스 해제 시 + 버튼 탭 시 | 빈 값 | "연락처를 입력해주세요" |
| phone | 포커스 해제 시 | 한국 휴대폰 형식 | "올바른 휴대폰 번호를 입력해주세요" |

### 에러 메시지 표시 스타일

| 속성 | 값 |
|------|-----|
| 텍스트 색상 | `#EF4444` (Error) |

| 위치 | 해당 입력 필드 바로 아래, 좌측 정렬 |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 역할 선택 | 고객 또는 샵 사장님 카드 탭 | 선택된 역할을 `role` 상태에 저장, 카드 시각적 전환, 버튼 텍스트 변경 | 선택 카드: 그린 테두리 + 연한 그린 배경, 체크 아이콘 표시. 고객 선택 시 버튼 "시작하기" + 스텝 인디케이터 숨김, 사장님 선택 시 버튼 "다음" + 스텝 인디케이터 표시 |
| 2 | 역할 변경 | 다른 역할 카드 탭 | 기존 선택 해제 + 새 역할 선택 + 버튼 텍스트 변경 | 카드 상태 스위칭, 스텝 인디케이터 표시/숨김 전환 |
| 3 | 이름 입력 | 이름 필드에 텍스트 입력 | `name` 상태 업데이트 | 실시간 텍스트 반영 |
| 4 | 연락처 입력 | 연락처 필드에 텍스트 입력 | `phone` 상태 업데이트 + 자동 하이픈 삽입 | "010-1234-5678" 형식 자동 포맷 |
| 5 | 시작하기 버튼 탭 (고객) | 고객 역할 선택 상태에서 "시작하기" 버튼 터치 | 전체 입력값 검증 → users 테이블 INSERT (auth.currentUser.id 사용) → members 매칭 | 성공: 고객 홈 화면 이동 / 실패: 에러 표시 |
| 6 | 다음 버튼 탭 (사장님) | 사장님 역할 선택 상태에서 "다음" 버튼 터치 | 전체 입력값 검증 → users 테이블 INSERT (auth.currentUser.id 사용) | 성공: 샵 등록 화면(2단계)으로 이동 / 실패: 에러 표시 |
| 7 | 뒤로가기 | 앱바 뒤로가기 아이콘 또는 시스템 백 | 로그인(소셜 로그인) 화면으로 이동 | 입력값은 유지하지 않음 |

### 프로필 설정 처리 흐름

```
시작하기/다음 버튼 탭
  ↓
클라이언트 전체 입력값 검증
  ↓
├── 검증 실패 → 첫 번째 에러 필드로 스크롤 + 에러 메시지 표시
│
└── 검증 성공
      ↓
    버튼을 로딩 상태로 전환 (스피너 표시)
      ↓
    users 테이블에 사용자 정보 INSERT
    (id: auth.currentUser.id, role, name, phone)
      ↓
    ├── 성공
    │     ↓
    │   phone 번호로 기존 members 매칭 시도
    │   (role이 customer인 경우만)
    │     ↓
    │   완료 토스트 표시 ("프로필 설정이 완료되었습니다!")
    │     ↓
    │   ├── role = "customer" → 고객 홈으로 이동
    │   └── role = "shop_owner" → 샵 등록 화면으로 이동 (2단계 진행)
    │
    └── 실패
          ↓
        버튼 로딩 해제
          ↓
        에러 타입에 따른 메시지 표시
          ├── 네트워크 오류 → "네트워크 연결을 확인해주세요"
          └── 기타 오류 → "프로필 설정에 실패했습니다. 다시 시도해주세요"
```

## 6. API 연동

### 6.1 데이터 조회

이 화면에서 초기 데이터 조회는 발생하지 않는다. 소셜 로그인 시 수신한 프로필 이름을 이름 필드의 기본값으로 사용한다.

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 사용자 정보 저장 | `users` | INSERT | `id` (auth.currentUser.id), `role`, `name`, `phone`, `created_at` | 시작하기/다음 버튼 탭 시 |
| 기존 회원 매칭 | `members` | UPDATE | `user_id = auth.currentUser.id` WHERE `phone = 입력한_phone` AND `user_id IS NULL` | users INSERT 성공 후 (customer만) |

**Supabase 쿼리 예시:**

```dart
// 1. users 테이블에 사용자 정보 저장 (소셜 로그인으로 이미 인증 완료)
final currentUser = supabase.auth.currentUser!;

await supabase.from('users').insert({
  'id': currentUser.id,
  'role': selectedRole,  // 'customer' 또는 'shop_owner'
  'name': name,
  'phone': phone,
  'created_at': DateTime.now().toIso8601String(),
});

// 2. 기존 members 매칭 (고객인 경우만)
if (selectedRole == 'customer') {
  await supabase
    .from('members')
    .update({'user_id': currentUser.id})
    .eq('phone', phone)
    .is_('user_id', null);
}
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 9. 비고

- **역할 선택은 가입 후 변경할 수 없다.** 역할 선택 카드 하단에 "가입 후 역할을 변경할 수 없습니다"라는 안내 텍스트를 `bodySmall` (12sp), `#9CA3AF` 색상으로 표시한다.
- **역할에 따라 버튼 텍스트가 동적으로 변경된다.** 고객 선택 시 "시작하기", 사장님 선택 시 "다음"으로 표시하여 2단계(샵 등록)가 이어짐을 암시한다.
- **샵 사장님으로 가입한 경우**, 프로필 완성 후 바로 **샵 등록 화면(2단계)**으로 이동한다. 샵 정보(이름, 주소, 소개글 등)를 먼저 등록해야 앱의 주요 기능을 사용할 수 있다.
- **고객으로 가입한 경우**, 프로필 완성 후 바로 고객 홈 화면으로 이동한다. 입력한 phone 번호와 일치하는 기존 `members` 레코드가 있으면 자동 매칭된다.
- **소셜 프로필에서 가져온 이름을 기본값으로 채워주되, 사용자가 수정 가능하다.** 소셜 로그인 제공자(카카오, 구글 등)에서 수신한 displayName을 이름 필드의 초기값으로 설정한다.
- 이 화면은 소셜 로그인으로 이미 인증이 완료된 신규 사용자를 위한 화면이다. Supabase Auth의 `signUp`은 호출하지 않으며, `auth.currentUser.id`를 그대로 users 테이블에 저장한다.
- 키보드가 올라올 때 화면이 잘리지 않도록 `SingleChildScrollView`로 감싸서 스크롤 가능하게 한다.
- 연락처 입력 시 숫자만 입력 가능하도록 `inputFormatters`를 적용하고, 자동으로 `010-XXXX-XXXX` 형식의 하이픈을 삽입한다.

### 접근성

- 뒤로가기 아이콘(`arrow_back_ios`)에 `semanticLabel: "뒤로가기"` 를 명시한다.
- 역할 선택 카드의 체크 아이콘(`check_circle`)에 `semanticLabel: "선택됨"` 을 명시한다.
- 텍스트 색상(`#1A1A2E`)과 배경(`#FFFFFF`)의 명암비는 15.4:1 로 WCAG AA 기준(4.5:1)을 충족한다.

### 다단계 프로세스 진행 표시

- 사장님 역할 선택 시, 앱바 하단에 **스텝 인디케이터**를 표시한다.
- 형식: `● 정보 입력 ─── ○ 샵 등록` (2단계 중 1단계 활성)
- 고객 역할 선택 시에는 스텝 인디케이터를 숨긴다 (1단계에서 완료되므로).

### Flutter 구현 가이드

- 전체 폼을 `Form` 위젯으로 감싸고 `GlobalKey<FormState>`를 사용하여 유효성 검증을 일괄 수행한다.
- 각 `TextFormField`에 `validator` 콜백을 정의하여 검증 규칙을 적용한다.
- 소셜 로그인 완료 시 전달받은 프로필 이름을 이름 `TextEditingController`의 초기값으로 설정한다.
