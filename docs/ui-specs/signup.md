# 회원가입 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `signup` |
| **화면 명** | 회원가입 |
| **목적** | 신규 사용자가 역할(고객/샵 사장님)을 선택하고 이메일, 비밀번호, 이름, 연락처를 입력하여 계정을 생성한다 |
| **사용자 역할** | 공통 |
| **진입 조건** | 미로그인 상태 (Supabase Auth 세션 없음) |

## 2. 레이아웃 구조

```
┌─────────────────────────────┐
│  ← 뒤로가기    "회원가입"     │  ← 앱바
├─────────────────────────────┤
│                             │
│  Step 1: 역할 선택           │
│  "어떤 사용자이신가요?"       │
│                             │
│  ┌───────────┐ ┌──────────┐│
│  │  🏸        │ │  🏪      ││
│  │  고객      │ │ 샵 사장님 ││
│  │  거트 작업  │ │ 작업 접수 ││
│  │  현황 확인  │ │ 및 관리  ││
│  └───────────┘ └──────────┘│
│                             │
├─────────────────────────────┤
│                             │
│  Step 2: 정보 입력           │
│                             │
│  ┌───────────────────────┐  │
│  │ 이메일 입력             │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 비밀번호 입력           │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 비밀번호 확인           │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 이름 입력              │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 연락처 입력             │  │
│  └───────────────────────┘  │
│                             │
│  ┌───────────────────────┐  │
│  │     가입하기 버튼       │  │
│  └───────────────────────┘  │
│                             │
│  이미 계정이 있으신가요?     │
│         로그인              │
│                             │
└─────────────────────────────┘
```

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 앱바 | 뒤로가기 아이콘 + "회원가입" 타이틀 | 뒤로가기 시 로그인 화면으로 이동 |
| 역할 선택 영역 | 고객/샵 사장님 역할 선택 카드 2개 | 가로로 나란히 배치, 반드시 하나 선택 |
| 정보 입력 영역 | 이메일, 비밀번호, 이름, 연락처 입력 폼 | 역할 선택 후 표시 (또는 항상 표시) |
| 가입 버튼 영역 | 가입하기 버튼 + 로그인 링크 | 모든 필수 필드 입력 후 활성화 |

## 4. 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 검증 규칙 | 설명 |
|--------|------|------|--------|-----------|------|
| role | Enum | Y | null | `customer` 또는 `shop_owner` 중 하나 선택 | 사용자 역할 |
| email | String | Y | "" | 이메일 형식, 중복 불가 | 로그인용 이메일 |
| password | String | Y | "" | 최소 6자 이상 | 로그인용 비밀번호 |
| passwordConfirm | String | Y | "" | password와 일치 | 비밀번호 확인 |
| name | String | Y | "" | 최소 2자, 최대 20자 | 사용자 이름 |
| phone | String | Y | "" | 한국 휴대폰 번호 형식 (010-XXXX-XXXX) | 연락처 |

### 검증 규칙 상세

| 필드 | 검증 시점 | 규칙 | 에러 메시지 |
|------|-----------|------|-------------|
| role | 가입 버튼 탭 시 | 선택 여부 확인 | "사용자 유형을 선택해주세요" |
| email | 포커스 해제 시 + 가입 버튼 탭 시 | 빈 값 | "이메일을 입력해주세요" |
| email | 포커스 해제 시 | 이메일 형식 | "올바른 이메일 형식을 입력해주세요" |
| email | 가입 버튼 탭 시 (서버 검증) | 중복 확인 | "이미 가입된 이메일입니다" |
| password | 포커스 해제 시 + 가입 버튼 탭 시 | 빈 값 | "비밀번호를 입력해주세요" |
| password | 포커스 해제 시 | 최소 6자 | "비밀번호는 6자 이상이어야 합니다" |
| passwordConfirm | 포커스 해제 시 + 가입 버튼 탭 시 | 빈 값 | "비밀번호 확인을 입력해주세요" |
| passwordConfirm | 포커스 해제 시 | password와 일치 | "비밀번호가 일치하지 않습니다" |
| name | 포커스 해제 시 + 가입 버튼 탭 시 | 빈 값 | "이름을 입력해주세요" |
| name | 포커스 해제 시 | 2~20자 | "이름은 2자 이상 20자 이하로 입력해주세요" |
| phone | 포커스 해제 시 + 가입 버튼 탭 시 | 빈 값 | "연락처를 입력해주세요" |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 역할 선택 | 고객 또는 샵 사장님 카드 탭 | 선택된 역할을 `role` 상태에 저장, 카드 시각적 전환 | 선택 카드: 그린 테두리 + 연한 그린 배경, 체크 아이콘 표시 |
| 2 | 역할 변경 | 다른 역할 카드 탭 | 기존 선택 해제 + 새 역할 선택 | 카드 상태 스위칭 |
| 3 | 이메일 입력 | 이메일 필드에 텍스트 입력 | `email` 상태 업데이트 | 실시간 텍스트 반영 |
| 4 | 비밀번호 입력 | 비밀번호 필드에 텍스트 입력 | `password` 상태 업데이트 | 마스킹된 텍스트 표시 |
| 5 | 비밀번호 표시 토글 | 눈 아이콘 탭 | `isPasswordVisible` 토글 | 비밀번호 텍스트 표시/마스킹 전환 |
| 6 | 비밀번호 확인 입력 | 비밀번호 확인 필드에 텍스트 입력 | `passwordConfirm` 상태 업데이트 | 마스킹된 텍스트 표시 |
| 7 | 이름 입력 | 이름 필드에 텍스트 입력 | `name` 상태 업데이트 | 실시간 텍스트 반영 |
| 8 | 연락처 입력 | 연락처 필드에 텍스트 입력 | `phone` 상태 업데이트 + 자동 하이픈 삽입 | "010-1234-5678" 형식 자동 포맷 |
| 9 | 가입하기 버튼 탭 | 가입하기 버튼 터치 | 전체 입력값 검증 → Supabase Auth 회원가입 → users 테이블 INSERT | 성공: 역할별 홈 화면 이동 / 실패: 에러 표시 |
| 10 | 로그인 링크 탭 | "로그인" 텍스트 터치 | 로그인 화면으로 이동 | 로그인 화면 표시 |
| 11 | 뒤로가기 | 앱바 뒤로가기 아이콘 또는 시스템 백 | 로그인 화면으로 이동 | 입력값은 유지하지 않음 |

### 회원가입 처리 흐름

```
가입하기 버튼 탭
  ↓
클라이언트 전체 입력값 검증
  ↓
├── 검증 실패 → 첫 번째 에러 필드로 스크롤 + 에러 메시지 표시
│
└── 검증 성공
      ↓
    버튼을 로딩 상태로 전환 (스피너 표시)
      ↓
    Supabase Auth signUp 호출 (email, password)
      ↓
    ├── 성공 (auth.user 생성됨)
    │     ↓
    │   users 테이블에 사용자 정보 INSERT
    │   (id, role, name, phone)
    │     ↓
    │   phone 번호로 기존 members 매칭 시도
    │   (role이 customer인 경우만)
    │     ↓
    │   가입 완료 토스트 표시 ("가입이 완료되었습니다!")
    │     ↓
    │   ├── role = "customer" → 고객 홈으로 이동
    │   └── role = "shop_owner" → 샵 등록 화면으로 이동
    │
    └── 실패
          ↓
        버튼 로딩 해제
          ↓
        에러 타입에 따른 메시지 표시
          ├── 이메일 중복 → "이미 가입된 이메일입니다"
          ├── 네트워크 오류 → "네트워크 연결을 확인해주세요"
          └── 기타 오류 → "가입에 실패했습니다. 다시 시도해주세요"
```

## 6. API 연동

### 6.1 데이터 조회

이 화면에서 초기 데이터 조회는 발생하지 않는다.

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 회원가입 | Supabase Auth | `signUp` | `email`, `password` | 가입하기 버튼 탭 시 |
| 사용자 정보 저장 | `users` | INSERT | `id`, `role`, `name`, `phone`, `created_at` | Auth 가입 성공 후 |
| 기존 회원 매칭 | `members` | UPDATE | `user_id = auth.uid` WHERE `phone = 입력한_phone` AND `user_id IS NULL` | users INSERT 성공 후 (customer만) |

**Supabase 쿼리 예시:**

```dart
// 1. Auth 회원가입
final authResponse = await supabase.auth.signUp(
  email: email,
  password: password,
);

// 2. users 테이블에 사용자 정보 저장
await supabase.from('users').insert({
  'id': authResponse.user!.id,
  'role': selectedRole,  // 'customer' 또는 'shop_owner'
  'name': name,
  'phone': phone,
  'created_at': DateTime.now().toIso8601String(),
});

// 3. 기존 members 매칭 (고객인 경우만)
if (selectedRole == 'customer') {
  await supabase
    .from('members')
    .update({'user_id': authResponse.user!.id})
    .eq('phone', phone)
    .is_('user_id', null);
}
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 9. 비고

- **역할 선택은 가입 후 변경할 수 없다.** 역할 선택 카드 하단에 "가입 후 역할을 변경할 수 없습니다"라는 안내 텍스트를 `bodySmall` (12sp), `#94A3B8` 색상으로 표시한다.
- **샵 사장님으로 가입한 경우**, 가입 완료 후 바로 **샵 등록 화면**으로 이동한다. 샵 정보(이름, 주소, 소개글 등)를 먼저 등록해야 앱의 주요 기능을 사용할 수 있다.
- **고객으로 가입한 경우**, 입력한 phone 번호와 일치하는 기존 `members` 레코드가 있으면 자동 매칭된다. 이를 통해 이전에 샵 사장님이 수동 등록한 작업 이력을 바로 확인할 수 있다.
- 키보드가 올라올 때 화면이 잘리지 않도록 `SingleChildScrollView`로 감싸서 스크롤 가능하게 한다.
- 연락처 입력 시 숫자만 입력 가능하도록 `inputFormatters`를 적용하고, 자동으로 `010-XXXX-XXXX` 형식의 하이픈을 삽입한다.
- 비밀번호 강도 표시기는 1단계에서는 제외한다. 추후 비밀번호 필드 아래에 강도 바(약함/보통/강함)를 추가할 수 있다.
- 이메일 인증(확인 메일 발송)은 1단계에서 생략하고, 추후 보안 강화 시 도입한다.
