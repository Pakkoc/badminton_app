## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 앱 실행 | 앱 아이콘 터치 | 스플래시 화면 표시 + Supabase 인증 세션 확인 | 아래 라우팅 로직에 따라 화면 전환 |

### 라우팅 로직

스플래시 화면은 최소 **1.5초** 동안 표시한 후, 인증 상태에 따라 자동 라우팅한다.

```
앱 실행
  ↓
스플래시 화면 표시 (최소 1.5초)
  ↓
Supabase Auth 세션 확인
  ↓
├── 세션 있음 (로그인 상태)
│     ↓
│   users 테이블 조회
│     ↓
│   ├── users 레코드 없음 → 프로필 설정 화면으로 이동
│   └── users 레코드 있음
│         ↓
│       shops 테이블에서 owner_id로 샵 보유 여부 확인
│         ↓
│       ├── 샵 보유 → 사장님 대시보드로 이동
│       └── 샵 미보유 → 고객 홈 화면으로 이동
│
└── 세션 없음 (미로그인)
      ↓
    로그인 화면으로 이동
```

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 인증 세션 확인 | Supabase Auth | `supabase.auth.currentSession` | Session 또는 null | 화면 진입 시 |
| 사용자 레코드 조회 | `users` | `id = 현재_사용자_id` | user 전체 또는 null | 세션이 유효할 때 |
| 샵 보유 여부 확인 | `shops` | `owner_id = 현재_사용자_id` | shop.id 또는 null | 사용자 레코드가 존재할 때 |

**Supabase 쿼리 예시:**

```dart
// 1. 세션 확인
final session = supabase.auth.currentSession;

// 2. 사용자 레코드 조회 (세션이 있을 때)
if (session != null) {
  final userId = session.user.id;
  final data = await supabase
    .from('users')
    .select()
    .eq('id', userId)
    .maybeSingle();

  if (data == null) {
    // → 프로필 설정 화면으로 이동
  }

  // 3. 샵 보유 여부 확인
  final shop = await supabase
    .from('shops')
    .select('id')
    .eq('owner_id', userId)
    .maybeSingle();

  if (shop != null) {
    // → 사장님 대시보드로 이동
  } else {
    // → 고객 홈 화면으로 이동
  }
}
```

### 6.2 데이터 변경

| API | 테이블 | 조건 | 변경 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| FCM 토큰 저장 | `users` | `id = 현재_사용자_id` | `fcm_token` | 세션이 유효하고 사용자 레코드가 존재할 때 (비동기, 실패해도 라우팅에 영향 없음) |

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 10. 비고

- 스플래시 화면은 최소 1.5초 표시하여 브랜드 인지도를 확보한다. 인증 확인이 더 빨리 완료되더라도 최소 시간을 유지한다.
- 네트워크가 불안정하여 세션 확인이 **5초** 이상 걸리면 타임아웃 처리하고 로그인 화면으로 이동한다.
- Flutter의 `flutter_native_splash` 패키지로 네이티브 스플래시를 먼저 표시하고, Flutter 엔진 로드 후 이 커스텀 스플래시 화면으로 전환하는 방식을 권장한다.
- 앱 버전 정보(`v1.0.0`)를 슬로건 아래에 작게 표시하는 것도 고려할 수 있다 (`bodySmall`, 12sp, `#9CA3AF`).

### 접근성

- 셔틀콕 아이콘에 `semanticLabel: "거트알림 앱 로고"` 를 명시한다.
- 로딩 인디케이터에 `semanticLabel: "인증 확인 중"` 를 명시하여 스크린 리더가 현재 상태를 알려줄 수 있도록 한다.
- 텍스트 색상(`#1A1A2E`)과 배경(`#FFFFFF`)의 명암비는 15.4:1 로 WCAG AA 기준(4.5:1)을 충족한다.
- 하단 슬로건 텍스트 색상(`#9CA3AF`)과 배경(`#FFFFFF`)의 명암비는 약 3.0:1 로, 보조 텍스트이므로 장식 용도로 허용하되, 핵심 정보가 아님을 전제한다.
