> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `splash` |
| **화면 명** | 스플래시 |
| **목적** | 앱 실행 시 브랜드 아이덴티티를 전달하고, 인증 상태를 확인하여 적절한 화면으로 자동 라우팅한다 |
| **사용자 역할** | 공통 |
| **진입 조건** | 없음 (앱 실행 시 최초 진입 화면) |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 앱 실행 | 앱 아이콘 터치 | 스플래시 화면 표시 + Supabase 인증 세션 확인 | 아래 라우팅 로직에 따라 화면 전환 |

### 라우팅 로직

스플래시 화면은 최소 **1.5초** 동안 표시한 후, 인증 상태에 따라 자동 라우팅한다.

```
앱 실행
  ↓
스플래시 화면 표시 (최소 1.5초)
  ↓
Supabase Auth 세션 확인
  ↓
├── 세션 있음 (로그인 상태)
│     ↓
│   users 테이블에서 role 조회
│     ↓
│   ├── role = "customer" → 고객 홈 화면으로 이동
│   └── role = "shop_owner" → 사장님 대시보드로 이동
│
└── 세션 없음 (미로그인)
      ↓
    로그인 화면으로 이동
```

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 인증 세션 확인 | Supabase Auth | `supabase.auth.currentSession` | Session 또는 null | 화면 진입 시 |
| 사용자 역할 조회 | `users` | `id = 현재_사용자_id` | `role` | 세션이 유효할 때 |

**Supabase 쿼리 예시:**

```dart
// 1. 세션 확인
final session = supabase.auth.currentSession;

// 2. 역할 조회 (세션이 있을 때)
if (session != null) {
  final user = await supabase
    .from('users')
    .select('role')
    .eq('id', session.user.id)
    .single();
}
```

### 6.2 데이터 변경

이 화면에서 데이터 변경은 발생하지 않는다.

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 인증 세션 확인 중 | 로고 + 로딩 스피너 표시 |
| **에러** | 네트워크 오류로 인증 확인 실패 | 로그인 화면으로 이동 (오프라인에서도 로그인 화면으로 폴백) |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `isCheckingAuth` | bool | true | 인증 세션 확인 중 여부 |

## 9. 애니메이션

| 요소 | 애니메이션 | 시간 | 설명 |
|------|-----------|------|------|
| 셔틀콕 아이콘 | 페이드인 + 스케일업 (0.8 → 1.0) | 500ms | 앱 실행 시 로고 등장 |
| 앱 이름 텍스트 | 페이드인 | 400ms, 200ms 딜레이 | 아이콘 등장 후 텍스트 페이드인 |
| 로딩 인디케이터 | 페이드인 | 300ms, 600ms 딜레이 | 텍스트 등장 후 스피너 표시 |
| 화면 전환 | 페이드아웃 | 300ms | 다음 화면으로 전환 시 |

## 10. 비고

- 스플래시 화면은 최소 1.5초 표시하여 브랜드 인지도를 확보한다. 인증 확인이 더 빨리 완료되더라도 최소 시간을 유지한다.
- 네트워크가 불안정하여 세션 확인이 **5초** 이상 걸리면 타임아웃 처리하고 로그인 화면으로 이동한다.
- Flutter의 `flutter_native_splash` 패키지로 네이티브 스플래시를 먼저 표시하고, Flutter 엔진 로드 후 이 커스텀 스플래시 화면으로 전환하는 방식을 권장한다.
- 앱 버전 정보(`v1.0.0`)를 슬로건 아래에 작게 표시하는 것도 고려할 수 있다 (`bodySmall`, 12sp, `#94A3B8`).
