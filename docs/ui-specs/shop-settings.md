# 샵 설정 — UI 화면 스펙

> 최종 수정일: 2026-02-23

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-shop-settings` |
| **화면 명** | 샵 설정 |
| **목적** | 샵 정보(이름, 주소, 연락처, 소개글)와 사장님 정보(이름, 연락처)를 확인하고 수정할 수 있도록 한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`인 사용자만 접근 가능, 샵 등록 완료 상태 |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 필드 수정 | 입력 필드 값 변경 | 변경 감지, 저장 버튼 활성화 | 저장 버튼 색상 `#22C55E`으로 변경 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 샵 정보 조회 | `shops` | `owner_id = 내_user_id` | name, address, latitude, longitude, phone, description | 화면 진입 시 |
| 주소 검색 | 카카오 주소 API (Daum 우편번호 서비스) + 카카오맵 Geocoding | 검색 키워드 | 주소 텍스트, 위도, 경도 | 주소 검색 시 |
| 사장님 정보 조회 | `users` | `id = 내_user_id` | name, phone | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
// 샵 정보 조회
final shop = await supabase
  .from('shops')
  .select('name, address, latitude, longitude, phone, description')
  .eq('owner_id', myUserId)
  .single();

// 사장님 정보 조회
final owner = await supabase
  .from('users')
  .select('name, phone')
  .eq('id', myUserId)
  .single();
```

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 샵 정보 수정 | `shops` | UPDATE | `{name, address, latitude, longitude, phone, description}` | 저장 버튼 탭 시 |
| 사장님 정보 수정 | `users` | UPDATE | `{name, phone}` | 저장 버튼 탭 시 |

**Supabase 쿼리 예시:**

```dart
// 샵 정보 수정
await supabase
  .from('shops')
  .update({
    'name': shopName,
    'address': shopAddress,
    'latitude': shopLatitude,
    'longitude': shopLongitude,
    'phone': shopPhone,
    'description': shopDescription,
  })
  .eq('owner_id', myUserId);

// 사장님 정보 수정
await supabase
  .from('users')
  .update({
    'name': ownerName,
    'phone': ownerPhone,
  })
  .eq('id', myUserId);
```

## 9. 비고

- 샵 정보와 사장님 정보는 **별도 섹션으로 시각적으로 구분**한다. 각 섹션에 아이콘과 타이틀을 부여하여 어떤 정보를 수정하는지 명확히 한다.
- **저장 버튼은 변경사항이 있을 때만 활성화**된다. 원래 값과 동일하면 비활성 상태(`#CBD5E1`)를 유지한다.
- 저장 시 샵 정보와 사장님 정보를 **한 번의 저장 버튼으로 동시에 저장**한다. 두 API를 병렬로 호출하여 응답 속도를 높인다.
- 변경사항이 있는 상태에서 **화면을 벗어나려 할 때 확인 다이얼로그**를 표시하여 실수로 변경사항을 잃지 않도록 한다.
- **주소 검색**: 주소는 직접 입력이 아닌 카카오 주소 API(Daum 우편번호 서비스) 웹뷰를 통해 검색으로만 입력할 수 있다. 선택된 주소는 카카오맵 SDK의 Geocoding API로 좌표(위도/경도)를 자동 변환한다.
- **지도 미리보기**: 주소 선택 후 해당 좌표를 카카오맵에 마커로 표시하여 사장님이 위치를 시각적으로 확인할 수 있도록 한다. 주소 미입력 시 안내 텍스트를 표시한다.
- 소개글은 **멀티라인 입력 필드**로 최대 3줄까지 표시하며, 최대 200자 제한을 둔다.
- 디자인 참고: 토스의 설정 화면, 배민의 가게 정보 관리 화면.

### 접근성
- 하단 네비게이션 아이콘에 접근성 라벨을 포함한다 (각 탭명과 동일).
- 섹션 아이콘(`store`, `person`)에 접근성 라벨을 포함한다 (`semanticLabel: "샵 정보"`, `semanticLabel: "사장님 정보"`).
- 에러 상태 필드는 색상(빨간 테두리) + 에러 텍스트 메시지를 병행하여, 색상만으로 정보를 전달하지 않는다.

### 피드백 보완
- 저장 버튼 탭 시 버튼에 로딩 인디케이터를 표시하고, 중복 탭을 방지한다 (`isSaving` 플래그 활용).
- 저장 성공 시 "설정이 저장되었습니다" 스낵바, 실패 시 에러 스낵바를 표시한다.
- 유효성 오류 시 해당 필드 바로 아래에 에러 메시지를 표시한다.

### Flutter 구현 참고
- 이 화면은 폼 입력이 있으므로 `StatefulWidget`을 사용하고, `Form` + `GlobalKey<FormState>`로 유효성 검증을 수행한다.
- 연락처 필드에는 `keyboardType: TextInputType.phone`을 적용한다.
