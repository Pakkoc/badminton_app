# 샵 등록 (사장님 가입 2단계) — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-shop-signup` |
| **화면 명** | 샵 등록 |
| **목적** | 사장님 회원가입 2단계로, 운영할 샵의 기본 정보(이름, 주소, 연락처, 소개글)를 등록하여 서비스 이용을 시작한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`, 샵 미등록 상태 (회원가입 직후 자동 이동) |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 샵 이름 입력 | 샵 이름 필드에 텍스트 입력 | `shopName` 상태 업데이트 | 실시간 텍스트 반영 |
| 2 | 주소 검색 | "검색" 버튼 또는 주소 필드 탭 | 주소 검색 바텀시트 표시 | 검색어 입력 → 결과 리스트 표시 |
| 3 | 주소 선택 | 검색 결과에서 주소 선택 | 주소 텍스트 + 좌표 자동 입력 | 주소 필드에 텍스트 표시, 지도 미리보기에 마커 표시 |
| 4 | 연락처 입력 | 연락처 필드에 숫자 입력 | `phone` 상태 업데이트 + 자동 하이픈 | 포맷팅된 연락처 표시 |
| 5 | 소개글 입력 | 소개글 필드에 텍스트 입력 | `description` 상태 업데이트 | 실시간 텍스트 반영 + 글자 수 카운터 갱신 |
| 6 | 등록 완료 탭 | "등록 완료" 버튼 터치 | 필수 필드 검증 → shops 테이블 INSERT | 성공: 대시보드로 이동 + "샵이 등록되었습니다!" 토스트 / 실패: 에러 표시 |

### 등록 처리 흐름

```
등록 완료 버튼 탭
  ↓
클라이언트 전체 입력값 검증
  ↓
├── 검증 실패 → 첫 번째 에러 필드로 스크롤 + 에러 메시지 표시
│
└── 검증 성공
      ↓
    버튼을 로딩 상태로 전환 (스피너 표시)
      ↓
    shops 테이블에 샵 정보 INSERT
    (owner_id, name, address, latitude, longitude, phone, description)
      ↓
    ├── 성공
    │     ↓
    │   "샵이 등록되었습니다!" 성공 토스트 표시
    │     ↓
    │   사장님 대시보드로 이동
    │
    └── 실패
          ↓
        버튼 로딩 해제
          ↓
        에러 스낵바 표시
          ├── 네트워크 오류 → "네트워크 연결을 확인해주세요"
          └── 기타 오류 → "등록에 실패했습니다. 다시 시도해주세요"
```

## 6. API 연동

### 6.1 데이터 조회

| API | 서비스 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 주소 검색 | 네이버 지도 Geocoding API | 검색 키워드 | 주소 목록 (주소 텍스트, 위도, 경도) | 주소 검색 시 |

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 샵 등록 | `shops` | INSERT | `{owner_id, name, address, latitude, longitude, phone, description}` | "등록 완료" 버튼 탭 시 |

**Supabase 쿼리 예시:**

```dart
// 샵 등록
await supabase.from('shops').insert({
  'owner_id': supabase.auth.currentUser!.id,
  'name': shopName,
  'address': address,
  'latitude': latitude,
  'longitude': longitude,
  'phone': phone,
  'description': description.isNotEmpty ? description : null,
  'created_at': DateTime.now().toIso8601String(),
});
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **초기** | 화면 진입 직후 | 빈 폼 표시, 지도 미리보기 비활성, 등록 버튼 비활성 |
| **입력 중** | 필드 입력 진행 중 | 실시간 검증 |

| **제출 중** | 등록 API 호출 중 | 등록 버튼에 로딩 인디케이터, 입력 비활성화 |
| **에러** | API 호출 실패 | 에러 스낵바 표시, 등록 버튼 재활성화 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|

## 9. 비고

- **필수 완료 화면**: 이 화면은 사장님 가입의 2단계로, 뒤로가기가 없다. 샵 등록을 완료해야 앱의 주요 기능(대시보드, 작업 관리 등)을 사용할 수 있다.
- **주소 검색**: 주소는 직접 입력이 아닌 검색을 통해서만 입력할 수 있다. 이는 정확한 좌표(위도/경도)를 확보하여 고객의 주변 샵 검색과 길찾기 기능에 활용하기 위함이다.
- **네이버 지도 Geocoding API**: 주소 검색 시 네이버 지도 Geocoding API를 호출하여 주소 텍스트를 좌표로 변환한다. API 키는 환경 변수로 관리한다.
- **지도 미리보기**: 주소 선택 후 해당 좌표를 지도에 마커로 표시하여 사장님이 위치를 시각적으로 확인할 수 있도록 한다.
- **키보드 처리**: `SingleChildScrollView`로 감싸서 키보드가 올라올 때 스크롤 가능하게 한다.
- **연락처 형식**: 일반 전화번호(02-XXX-XXXX), 휴대폰(010-XXXX-XXXX) 등 다양한 형식을 허용한다.
