# 샵 등록 (사장님 가입 2단계) — UI 화면 스펙

> 최종 수정일: 2026-02-22

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-shop-signup` |
| **화면 명** | 샵 등록 |
| **목적** | 사장님 가입 2단계로, 운영할 샵의 기본 정보(이름, 주소, 연락처, 소개글)를 등록하여 서비스 이용을 시작한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`, 샵 미등록 상태 (프로필 설정 직후 자동 이동) |

**주소 검색 방식:**
- "검색" 버튼 탭 또는 입력 필드 탭 시 주소 검색 바텀시트를 표시한다.
- 바텀시트 내부에 카카오 주소 API(Daum 우편번호 서비스) 웹뷰가 로드된다.
- 카카오맵 SDK의 Geocoding으로 좌표(위도, 경도)를 자동 변환한다.
- 검색 결과 선택 시 주소 텍스트 + 좌표가 자동 입력되고 바텀시트가 닫힌다.

**주소 검색 바텀시트 스타일:** (Pencil 디자인: `Address Search BottomSheet` 프레임 참조)

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 샵 이름 입력 | 샵 이름 필드에 텍스트 입력 | `shopName` 상태 업데이트 | 실시간 텍스트 반영 |
| 2 | 주소 검색 | "검색" 버튼 또는 주소 필드 탭 | 주소 검색 바텀시트 표시 | 검색어 입력 → 결과 리스트 표시 |
| 3 | 주소 선택 | 검색 결과에서 주소 선택 | 주소 텍스트 + 좌표 자동 입력 | 주소 필드에 텍스트 표시, 지도 미리보기에 마커 표시 |
| 4 | 연락처 입력 | 연락처 필드에 숫자 입력 | `phone` 상태 업데이트 + 자동 하이픈 | 포맷팅된 연락처 표시 |
| 5 | 소개글 입력 | 소개글 필드에 텍스트 입력 | `description` 상태 업데이트 | 실시간 텍스트 반영 + 글자 수 카운터 갱신 |
| 6 | 등록 완료 탭 | "등록 완료" 버튼 터치 | 필수 필드 검증 → shops 테이블 INSERT | 성공: 대시보드로 이동 + "샵이 등록되었습니다!" 토스트 / 실패: 에러 표시 |

### 등록 처리 흐름

```
등록 완료 버튼 탭
  ↓
클라이언트 전체 입력값 검증
  ↓
├── 검증 실패 → 첫 번째 에러 필드로 스크롤 + 에러 메시지 표시
│
└── 검증 성공
      ↓
    버튼을 로딩 상태로 전환 (스피너 표시)
      ↓
    shops 테이블에 샵 정보 INSERT
    (owner_id, name, address, latitude, longitude, phone, description)
      ↓
    ├── 성공
    │     ↓
    │   "샵이 등록되었습니다!" 성공 토스트 표시
    │     ↓
    │   사장님 대시보드로 이동
    │
    └── 실패
          ↓
        버튼 로딩 해제
          ↓
        에러 스낵바 표시
          ├── 네트워크 오류 → "네트워크 연결을 확인해주세요"
          └── 기타 오류 → "등록에 실패했습니다. 다시 시도해주세요"
```

## 6. API 연동

### 6.1 데이터 조회

| API | 서비스 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 주소 검색 | 카카오 주소 API (Daum 우편번호 서비스) + 카카오맵 Geocoding | 검색 키워드 | 주소 목록 (주소 텍스트, 위도, 경도) | 주소 검색 시 |

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 샵 등록 | `shops` | INSERT | `{owner_id, name, address, latitude, longitude, phone, description}` | "등록 완료" 버튼 탭 시 |

**Supabase 쿼리 예시:**

```dart
// 샵 등록
await supabase.from('shops').insert({
  'owner_id': supabase.auth.currentUser!.id,
  'name': shopName,
  'address': address,
  'latitude': latitude,
  'longitude': longitude,
  'phone': phone,
  'description': description.isNotEmpty ? description : null,
  'created_at': DateTime.now().toIso8601String(),
});
```

### 6.3 실시간 구독

## 9. 비고

- **필수 완료 화면**: 이 화면은 사장님 가입의 2단계로, 뒤로가기가 없다. 샵 등록을 완료해야 앱의 주요 기능(대시보드, 작업 관리 등)을 사용할 수 있다.
- **주소 검색**: 주소는 직접 입력이 아닌 검색을 통해서만 입력할 수 있다. 이는 정확한 좌표(위도/경도)를 확보하여 고객의 주변 샵 검색과 길찾기 기능에 활용하기 위함이다.
- **카카오 주소 API + 카카오맵 SDK**: 주소 검색 시 카카오 주소 API(Daum 우편번호 서비스) 웹뷰로 주소를 검색하고, 카카오맵 SDK의 Geocoding API를 통해 좌표로 변환한다. API 키는 환경 변수로 관리한다.
- **지도 미리보기**: 주소 선택 후 해당 좌표를 지도에 마커로 표시하여 사장님이 위치를 시각적으로 확인할 수 있도록 한다.
- **키보드 처리**: `SingleChildScrollView`로 감싸서 키보드가 올라올 때 스크롤 가능하게 한다.
- **연락처 형식**: 일반 전화번호(02-XXX-XXXX), 휴대폰(010-XXXX-XXXX) 등 다양한 형식을 허용한다.

### 접근성

- 주소 검색 버튼에 `semanticLabel: "주소 검색"` 을 명시한다.
- 지도 미리보기에 `Semantics(label: "샵 위치 지도 미리보기")` 를 적용한다.
- 지도 미리보기의 마커에 `semanticLabel: "샵 위치"` 를 명시한다.
- 텍스트 색상(`#0F172A`)과 배경(`#FFFFFF`)의 명암비는 15.4:1 로 WCAG AA 기준(4.5:1)을 충족한다.
- 라벨 텍스트 색상(`#475569`)과 배경(`#FFFFFF`)의 명암비는 약 7.2:1 로 AA 기준 충족.

### 다단계 프로세스 진행 표시

- 앱바 하단에 **스텝 인디케이터**를 표시하여 현재 2단계 중 2단계임을 나타낸다.
- 형식: `✓ 정보 입력 ─── ● 샵 등록` (2단계 중 2단계 활성)

| 속성 | 값 |
|------|-----|
| 완료 단계 아이콘 | `check_circle`, 색상 `#16A34A` |
| 활성 단계 색상 | `#16A34A` (Court Green) |
| 연결선 색상 | `#16A34A` (완료 구간) |
| 텍스트 크기 | 12sp (`bodySmall`) |
| 위치 | 앱바 아래, 안내 영역 위, 수평 중앙 정렬 |
| 상하 패딩 | 12px |

### 로딩 상태 (스켈레톤)

- 주소 검색 시 검색 결과 로딩 중에는 바텀시트 내에 `CircularProgressIndicator`(색상 `#16A34A`)를 표시한다.
- 지도 미리보기 로딩 중에는 지도 영역에 shimmer 효과(배경 `#E2E8F0` → `#F1F5F9`, 1500ms 반복)를 표시한다.

### Flutter 구현 가이드

- 전체 폼을 `Form` 위젯으로 감싸고 `GlobalKey<FormState>`를 사용하여 유효성 검증을 일괄 수행한다.
- 각 `TextFormField`에 `validator` 콜백을 정의하여 검증 규칙을 적용한다.
