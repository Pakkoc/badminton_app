## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `owner-member-list` |
| **화면 명** | 회원 관리 |
| **목적** | 샵에 등록된 회원 목록을 조회하고, 이름 또는 연락처로 검색하여 회원 상세 정보에 빠르게 접근할 수 있도록 한다 |
| **사용자 역할** | shop_owner |
| **진입 조건** | 로그인 필수, role이 `shop_owner`인 사용자만 접근 가능, 샵 등록 완료 상태 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 회원 목록 조회 | `members` LEFT JOIN `orders` | `shop_id = 내_샵_id` ORDER BY 최근 방문일 DESC | member.*, 최근 방문일 | 화면 진입 시, Pull-to-refresh 시 |
| 회원 검색 | `members` | `shop_id = 내_샵_id` AND (`name ILIKE 검색어` OR `phone ILIKE 검색어`) | member.* | 검색어 입력 시 (debounce 300ms) |
| 회원 수 조회 | `members` | `shop_id = 내_샵_id` | COUNT | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
// 회원 목록 조회 (최근 방문일 포함)
final members = await supabase
  .from('members')
  .select('*, orders(created_at)')
  .eq('shop_id', myShopId)
  .order('created_at', ascending: false);

// 회원 검색
final searchResult = await supabase
  .from('members')
  .select('*')
  .eq('shop_id', myShopId)
  .or('name.ilike.%$query%,phone.ilike.%$query%');

// 회원 수 조회
final count = await supabase
  .from('members')
  .select('id', const FetchOptions(count: CountOption.exact))
  .eq('shop_id', myShopId);
```

## 9. 비고

- 회원 목록은 **최근 방문일 기준 내림차순**으로 정렬한다. 최근 방문일은 해당 회원의 가장 최근 `orders.created_at` 기준이다. 방문 기록이 없는 회원은 목록 하단에 표시한다.
- 검색은 **실시간 검색(debounce 300ms)** 방식으로 구현한다. 이름과 연락처 모두에 대해 부분 일치(ILIKE) 검색을 수행한다.
- 회원 수는 앱바에 항상 표시하여 샵의 전체 회원 규모를 한눈에 파악할 수 있도록 한다. 검색 중에는 검색 결과 건수로 변경된다.
- FAB는 스크롤 시 잠시 축소되었다가 스크롤 멈추면 다시 표시되는 동작을 적용한다.
- 회원 카드에서 회원 상세로 이동하면 선호 거트, 라켓 정보, 방문 이력 등 상세 정보를 확인할 수 있다.
- 디자인 참고: 토스의 연락처 리스트, 배민의 회원 관리 화면.
- **텍스트 명암비**: 본문 텍스트(`#475569`)와 배경(`#FFFFFF`)의 명암비는 WCAG AA 기준 4.5:1 이상을 충족해야 한다.
- **터치 타겟**: 모든 터치 가능 요소(회원 카드, FAB, 검색바 초기화 버튼, 하단 네비게이션)의 터치 영역은 최소 44x44px을 보장한다. 인접 터치 타겟 간 최소 8px 간격을 유지한다.
