# 로그인 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `login` |
| **화면 명** | 로그인 |
| **목적** | 기존 사용자가 이메일/비밀번호로 로그인하여 앱에 접근한다. 미가입 사용자는 회원가입 화면으로 이동할 수 있다 |
| **사용자 역할** | 공통 |
| **진입 조건** | 미로그인 상태 (Supabase Auth 세션 없음) |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 이메일 입력 | 이메일 필드에 텍스트 입력 | 입력값을 `email` 상태에 저장 | 실시간 텍스트 반영 |
| 2 | 비밀번호 입력 | 비밀번호 필드에 텍스트 입력 | 입력값을 `password` 상태에 저장 | 마스킹된 텍스트 표시 |
| 3 | 비밀번호 표시 토글 | 비밀번호 필드 우측 눈 아이콘 탭 | `obscureText` 토글 | 비밀번호 텍스트 표시/마스킹 전환 |
| 4 | 로그인 버튼 탭 | 로그인 버튼 터치 | 입력값 검증 → Supabase Auth 로그인 호출 | 성공: 역할별 홈 화면 이동 / 실패: 에러 메시지 표시 |
| 5 | 비밀번호 찾기 탭 | "비밀번호를 잊으셨나요?" 터치 | 비밀번호 재설정 화면으로 이동 | 비밀번호 재설정 화면 표시 |
| 6 | 회원가입 버튼 탭 | 회원가입 버튼 터치 | 회원가입 화면으로 이동 | 회원가입 화면 표시 |
| 7 | 키보드 완료 | 비밀번호 필드에서 키보드 "완료" 탭 | 로그인 실행 (인터랙션 #4와 동일) | 로그인 시도 |

### 로그인 처리 흐름

```
로그인 버튼 탭
  ↓
클라이언트 입력값 검증
  ↓
├── 검증 실패 → 해당 필드에 에러 메시지 표시
│
└── 검증 성공
      ↓
    버튼을 로딩 상태로 전환 (스피너 표시)
      ↓
    Supabase Auth signInWithPassword 호출
      ↓
    ├── 성공
    │     ↓
    │   users 테이블에서 role 조회
    │     ↓
    │   ├── role = "customer" → 고객 홈으로 이동
    │   └── role = "shop_owner" → 사장님 대시보드로 이동
    │
    └── 실패
          ↓
        버튼 로딩 해제
          ↓
        에러 타입에 따른 메시지 표시
          ├── 이메일/비밀번호 불일치 → "이메일 또는 비밀번호가 올바르지 않습니다"
          ├── 네트워크 오류 → "네트워크 연결을 확인해주세요"
          └── 기타 오류 → "로그인에 실패했습니다. 다시 시도해주세요"
```

### 에러 스낵바 스타일

| 속성 | 값 |
|------|-----|
| 배경색 | `#FEE2E2` (Error Background) |
| 텍스트 색상 | `#EF4444` (Error) |
| 좌측 아이콘 | `error_outline`, 색상 `#EF4444` |
| 모서리 둥글기 | 8px |

## 6. API 연동

### 6.1 데이터 조회

이 화면에서 초기 데이터 조회는 발생하지 않는다.

### 6.2 데이터 변경

| API | 테이블 | 동작 | 파라미터 | 호출 시점 |
|-----|--------|------|----------|-----------|
| 로그인 | Supabase Auth | `signInWithPassword` | `email`, `password` | 로그인 버튼 탭 시 |
| 역할 조회 | `users` | SELECT | `id = auth.uid` | 로그인 성공 후 |

**Supabase 쿼리 예시:**

```dart
// 1. 로그인
final response = await supabase.auth.signInWithPassword(
  email: email,
  password: password,
);

// 2. 역할 조회
final user = await supabase
  .from('users')
  .select('role')
  .eq('id', response.user!.id)
  .single();

// 3. 역할별 라우팅
if (user['role'] == 'customer') {
  // 고객 홈으로 이동
} else if (user['role'] == 'shop_owner') {
  // 사장님 대시보드로 이동
}
```

### 6.3 실시간 구독

이 화면에서 실시간 구독은 사용하지 않는다.

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **기본** | 화면 진입 시 | 빈 입력 필드 + 비활성 로그인 버튼 |
| **입력 중** | 이메일 또는 비밀번호 필드에 포커스 | 포커스된 필드 테두리 `#16A34A` |
| **유효** | 이메일, 비밀번호 모두 입력됨 | 로그인 버튼 활성 (`#16A34A` 배경) |
| **로딩** | 로그인 API 호출 중 | 로그인 버튼에 CircularProgressIndicator 표시, 버튼 비활성 |
| **에러** | 로그인 실패 | 에러 스낵바 표시 + 입력 필드 유지 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `email` | String | "" | 이메일 입력값 |
| `password` | String | "" | 비밀번호 입력값 |
| `isPasswordVisible` | bool | false | 비밀번호 표시 여부 |
| `isLoading` | bool | false | 로그인 API 호출 중 여부 |
| `emailError` | String? | null | 이메일 검증 에러 메시지 |
| `passwordError` | String? | null | 비밀번호 검증 에러 메시지 |

## 9. 비고

- 로그인 화면에서 **뒤로가기 (시스템 백 버튼)** 를 누르면 앱을 종료한다 (스플래시로 돌아가지 않음).
- 이메일 필드에 포커스가 자동으로 들어가도록 `autofocus: true`를 설정한다.
- 키보드가 올라올 때 화면이 잘리지 않도록 `SingleChildScrollView`로 감싸서 스크롤 가능하게 한다.
- 로그인 버튼의 활성/비활성 상태는 이메일과 비밀번호가 모두 1자 이상 입력되었을 때를 기준으로 한다 (형식 검증은 버튼 탭 시 수행).
- 소셜 로그인 (카카오, 구글 등)은 1단계에서는 제외하고, 추후 확장 시 로그인 버튼과 회원가입 유도 영역 사이에 "또는" 구분선 + 소셜 버튼을 추가한다.

### 접근성

- 비밀번호 표시/숨기기 토글 아이콘에 `semanticLabel`을 명시한다: 표시 상태 → `"비밀번호 숨기기"`, 숨김 상태 → `"비밀번호 표시"`.
- 이메일 입력 필드의 좌측 아이콘(`mail_outline`)에 `semanticLabel: "이메일"` 을 명시한다.
- 비밀번호 입력 필드의 좌측 아이콘(`lock_outline`)에 `semanticLabel: "비밀번호"` 을 명시한다.
- 텍스트 색상(`#0F172A`)과 배경(`#FFFFFF`)의 명암비는 15.4:1 로 WCAG AA 기준(4.5:1)을 충족한다.
- 에러 텍스트 색상(`#EF4444`)과 배경(`#FFFFFF`)의 명암비는 약 4.0:1 로 AA Large Text 기준은 충족하나, AA Normal Text(4.5:1)에는 미달할 수 있다. 에러 아이콘을 병행하여 보완한다.
