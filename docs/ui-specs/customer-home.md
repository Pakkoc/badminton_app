# 고객 홈 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-home` |
| **화면 명** | 고객 홈 |
| **목적** | 현재 진행 중인 거트 작업 목록을 한눈에 확인하고, 각 작업의 실시간 상태(접수됨/작업중/완료)를 파악한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능 |

```
┌──────────────────────────────────┐

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 앱 타이틀("거트알림")과 알림 아이콘 표시 | 알림 아이콘에 읽지 않은 알림 수 뱃지 표시. 앱바 배경 `#FFFFFF`, 하단 테두리 `#E2E8F0` 0.5px |
| 진행 중 요약 카드 | 현재 접수됨/작업중 상태의 작업 건수를 요약 표시 | 완료 건은 포함하지 않음. 진행 중 작업이 0건이면 카드 자체를 숨김 |
| 내 작업 섹션 | 진행 중인 작업 카드를 세로 스크롤 목록으로 표시 | 최신 접수 순(created_at DESC) 정렬. Pull-to-refresh 지원 |
| 하단 네비게이션 | 5개 탭으로 주요 화면 이동 | 현재 탭(홈)은 `#16A34A`로 활성 상태 강조 |

### 3.3 작업 카드 (Order Card)

```

│                                         │
│  YONEX AX99 Pro               ● 작업중  │  ← Row: 라켓 모델 + 상태 뱃지

│                                         │  ← SizedBox(8px)
│  🏪 OO 거트 스트링샵                     │  ← 샵 이름 (아이콘 + 텍스트)
│  📅 2시간 전                             │  ← 접수 시간 (아이콘 + 상대시간)

└─────────────────────────────────────────┘

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| racket_model | String | orders.racket_model | 텍스트 그대로 | 라켓 모델명. 카드 좌측 상단에 굵게 표시 |
| gut_name | String | orders.gut_name | 텍스트 그대로 | 거트명 |
| tension | String | orders.tension | 텍스트 (예: "25lbs", "25/27lbs") | 텐션값. 거트명과 "\|"로 구분하여 한 줄에 표시 |
| status | Enum | orders.status | StatusBadge 컴포넌트 | `received` / `in_progress` / `completed` |
| shop_name | String | shops.name | 텍스트, 앞에 `storefront` 아이콘 | 작업을 맡긴 샵 이름 |
| created_at | Timestamp | orders.created_at | 상대시간 (기본) / 절대시간 (24시간 이후) | 접수 일시 |
| shop_latitude | Double | shops.latitude | 내부 사용 (길찾기용) | 샵 위도 |
| shop_longitude | Double | shops.longitude | 내부 사용 (길찾기용) | 샵 경도 |

### 시간 표시 규칙

| 조건 | 표시 형식 | 예시 |
|------|-----------|------|
| 1분 미만 | "방금 전" | 방금 전 |
| 1분 이상 ~ 60분 미만 | "N분 전" | 23분 전 |
| 1시간 이상 ~ 24시간 미만 | "N시간 전" | 2시간 전 |
| 24시간 이상 ~ 7일 미만 | "N일 전" | 3일 전 |
| 7일 이상 | "MM/DD HH:mm" | 02/15 14:30 |

| 6 | 빈 상태 버튼 탭 | "주변 샵 검색하기" 버튼 터치 | 샵 검색 화면으로 이동 | `customer-shop-search` 화면 전환 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 내 활성 작업 목록 조회 | `orders` JOIN `members` JOIN `shops` | `members.user_id = 현재_사용자_id` AND (`orders.status IN ('received', 'in_progress')` OR (`orders.status = 'completed'` AND `orders.updated_at > NOW() - 24시간`)) | orders.*, shops.name, shops.latitude, shops.longitude | 화면 진입 시, Pull-to-refresh 시 |
| 진행 중 건수 조회 | `orders` JOIN `members` | `members.user_id = 현재_사용자_id` AND `orders.status IN ('received', 'in_progress')` | COUNT by status | 화면 진입 시 (작업 목록 조회와 함께) |

**Supabase 쿼리 예시:**

```dart
// 내 활성 작업 목록 조회
final orders = await supabase
  .from('orders')
  .select('''
    *,
    members!inner(user_id, shop_id),
    shops!inner(name, latitude, longitude)
  ''')
  .eq('members.user_id', currentUserId)
  .in_('status', ['received', 'in_progress', 'completed'])
  .order('created_at', ascending: false);

// 완료 건은 클라이언트에서 24시간 이내만 필터링
final activeOrders = orders.where((order) {
  if (order['status'] == 'completed') {
    final updatedAt = DateTime.parse(order['updated_at']);
    return DateTime.now().difference(updatedAt).inHours < 24;
  }
  return true;
}).toList();
```

이 화면에서 데이터 변경은 발생하지 않는다. (조회 전용 화면)

## 9. 비고

### 완료 작업 표시 정책
- 완료된 작업은 홈 화면에 **24시간 동안만** 표시한다. 이후에는 "작업 이력"(하단 "이력" 탭) 화면에서만 확인 가능하다.
- 24시간 기준은 `orders.updated_at` (상태가 완료로 변경된 시점) 기준이다.

### 딥링크 처리
- 푸시 알림으로 앱 진입 시 `order_id`가 전달되면 홈을 거치지 않고 바로 작업 상세 화면(`customer-order-detail`)으로 딥링크한다.
- 앱이 이미 실행 중인 경우에도 동일하게 작업 상세 화면으로 이동한다.

### 길찾기 아이콘
- 작업 카드에서 완료 상태인 경우에만 카드 우측 하단에 "길찾기" 아이콘 버튼(`directions`, `#16A34A`)을 추가 표시한다.
- 탭 시 네이버 지도 앱의 길찾기를 호출한다 (`nmap://route/...`).
- 네이버 지도 앱 미설치 시 웹 브라우저로 네이버 지도 웹 버전을 폴백 호출한다.

### 실시간 구독 정책
- 실시간 구독은 앱이 **포그라운드** 상태일 때만 활성화한다.
- 백그라운드 상태에서는 FCM 푸시 알림에 의존한다.
- 앱이 포그라운드로 복귀 시 자동으로 데이터를 재조회(refresh)한다.

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 상태 뱃지는 색상만으로 구분하지 않고 아이콘 + 텍스트를 병행한다.
- 카드의 탭 가능 영역은 카드 전체이다 (InkWell/GestureDetector로 감싸기).

### 디자인 레퍼런스
- **토스**: 카드 기반의 깔끔한 레이아웃, 정보 계층 구조
- **배민**: 둥근 모서리(16px), 부드러운 탭 피드백, 친근한 빈 상태 일러스트
- **카카오택시**: 작업중 상태의 펄스 애니메이션, 실시간 상태 추적 UX

### 접근성 보완

- 알림 아이콘 버튼에 `semanticLabel: "알림"` 을 명시한다. 읽지 않은 알림이 있을 때는 `"읽지 않은 알림 N건"` 으로 동적 변경한다.
- 길찾기 아이콘 버튼에 `semanticLabel: "길찾기"` 을 명시한다.
- 하단 네비게이션 각 탭에 `semanticLabel`을 명시한다 (예: `"홈 탭, 현재 선택됨"`).
- 빈 상태의 일러스트 이미지에 `semanticLabel: "진행 중인 작업 없음"` 을 명시한다.
