# 작업 이력 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-order-history` |
| **화면 명** | 작업 이력 |
| **목적** | 과거 완료된 거트 작업 목록을 최신순으로 확인하여 이전 작업 내역(샵 정보, 완료일)을 참조할 수 있도록 한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능 |

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| shop_name | String | shops.name | 텍스트 | 작업한 샵 이름 |
| completed_at | Timestamp | orders.updated_at | "YYYY.MM.DD 완료" | 작업 완료일 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 완료된 작업 목록 조회 | `orders` JOIN `shops` | `member.user_id = 내_user_id` AND `status = 'completed'` ORDER BY `updated_at` DESC | order.*, shop.name | 화면 진입 시, Pull-to-refresh 시, 페이지네이션 시 |

**Supabase 쿼리 예시:**

```dart
// 완료된 작업 목록 (페이지네이션)
final completedOrders = await supabase
  .from('orders')
  .select('*, shops(name), members!inner(user_id)')
  .eq('members.user_id', myUserId)
  .eq('status', 'completed')
  .order('updated_at', ascending: false)
  .range(offset, offset + pageSize - 1);
```

### 6.2 데이터 변경

이 화면에서는 데이터 변경이 없다.

### 6.3 실시간 구독

| 구독 대상 | 테이블 | 조건 | 이벤트 | 처리 |
|-----------|--------|------|--------|------|
| 내 작업 완료 | `orders` | `member.user_id = 내_user_id` AND `status = 'completed'` | UPDATE | 목록 상단에 새 완료 작업 추가 |

## 9. 비고

- 작업 목록은 **완료일 기준 최신순**으로 정렬한다. 완료일은 `orders.updated_at` (상태가 `completed`로 변경된 시점) 기준이다.
- 페이지네이션은 **무한 스크롤** 방식으로 구현한다. 한 페이지당 20건씩 로드하며, 목록 하단에 도달하면 자동으로 다음 페이지를 로드한다.
- 작업 카드에는 **샵 이름을 타이틀로** 강조 표시하여, 어떤 샵에서 작업했는지 빠르게 식별할 수 있도록 한다.
- 작업 카드 탭 시 작업 상세 화면으로 이동하며, 상세 화면에서는 상태 타임라인(접수 → 작업중 → 완료)과 전체 작업 정보를 확인할 수 있다.
- 디자인 참고: 토스의 거래 내역 리스트, 배민의 주문 이력 화면.

### 접근성
- 모든 터치 타겟은 최소 48x48px을 보장한다.
- 작업 카드의 완료 아이콘(`check_circle`)은 색상(`#22C55E`) + 아이콘 형태로 구분하여, 색상만으로 정보를 전달하지 않는다.
- 하단 네비게이션의 각 탭 아이콘에 시맨틱 라벨(탭 이름)을 포함한다.

### 성공/실패 피드백
- 에러 재시도 시 로딩 상태로 전환 후 API 재호출. 성공 시 목록 표시, 실패 시 에러 상태 재표시.
- Pull-to-refresh 완료 시 목록이 갱신되며, 실패 시 "새로고침에 실패했습니다" 스낵바 표시.

### Flutter 구현 참고
- 이 화면은 실시간 구독(stream) + 페이지네이션을 사용하므로 `StatefulWidget`으로 구현한다.
- 목록은 `ListView.builder`로 구현하여 대량의 완료 작업도 효율적으로 렌더링한다.
