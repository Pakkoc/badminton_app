# Model Selection Guide

에이전트의 모델 선택 기준. **최종 품질**을 기준으로 선택한다.

---

## 선택 매트릭스

### opus 선택 기준

| 작업 특성 | 근거 |
|-----------|------|
| 복잡한 추론 체인 필요 | 다단계 논리 추론에서 정확도 우위 |
| 창의적 생성 (카피, 디자인, 설계) | 더 풍부하고 다양한 결과물 |
| 도메인 깊이 필요 (아키텍처, 전략) | 전문적 지식 활용 우위 |
| 적대적 분석·비판 | 비판적 사고의 깊이 우위 |
| 번역·문체 변환 | 언어적 뉘앙스 포착 우위 |
| 장문 산출물 (보고서, 문서) | 구조적 일관성 유지 우위 |
| 에이전트 간 조율 필요 | 복잡한 컨텍스트 이해 우위 |

### sonnet 선택 기준

| 작업 특성 | 근거 |
|-----------|------|
| 기계적 검증·반복 | 결정론적 작업에 충분한 성능 |
| 데이터 수집·정찰 | 구조화된 수집 작업 |
| 포맷 변환·정리 | 단순 변환 작업 |
| 코드 포맷팅·린팅 | 규칙 기반 작업 |
| 의존성 검증 | 체크리스트 기반 작업 |

---

## 결정 플로우차트

```
이 에이전트의 핵심 작업이 무엇인가?
│
├─ 판단·추론·창의성이 필요 → opus
│   ├─ 아키텍처 설계 → opus
│   ├─ 카피라이팅 → opus
│   ├─ UX 리서치 분석 → opus
│   ├─ 적대적 리뷰 → opus
│   └─ 전략 수립 → opus
│
└─ 규칙·패턴 따르기가 핵심 → sonnet
    ├─ 데이터 수집 → sonnet
    ├─ 형식 변환 → sonnet
    ├─ 체크리스트 검증 → sonnet
    └─ 반복적 처리 → sonnet
```
