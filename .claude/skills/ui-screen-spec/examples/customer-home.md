# 고객 홈 — UI 화면 스펙

> 최종 수정일: 2026-02-20

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면 ID** | `customer-home` |
| **화면 명** | 고객 홈 |
| **목적** | 현재 진행 중인 거트 작업 목록을 한눈에 확인하고, 각 작업의 실시간 상태를 파악한다 |
| **사용자 역할** | customer |
| **진입 조건** | 로그인 필수, role이 `customer`인 사용자만 접근 가능 |

---

## 2. 레이아웃 구조

```
┌─────────────────────────────┐
│  상단 앱바                    │
│  "거트알림" 로고 + 알림 아이콘  │
├─────────────────────────────┤
│  진행 중인 작업 요약 카드       │
│  (접수 N건 · 작업중 N건)      │
├─────────────────────────────┤
│                             │
│  작업 목록 (스크롤 영역)       │
│  ┌───────────────────────┐  │
│  │ 작업 카드 1            │  │
│  │ 샵명 · 거트명 · 상태    │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ 작업 카드 2            │  │
│  └───────────────────────┘  │
│          ...                │
│                             │
├─────────────────────────────┤
│  하단 네비게이션 바            │
│  홈 | 샵검색 | QR | 이력 | MY │
└─────────────────────────────┘
```

### 섹션 설명

| 섹션 | 설명 | 비고 |
|------|------|------|
| 상단 앱바 | 앱 로고와 알림 아이콘 표시 | 알림 아이콘에 읽지 않은 알림 수 뱃지 표시 |
| 진행 중 요약 카드 | 현재 접수/작업중 상태의 작업 건수를 요약 표시 | 완료 건은 포함하지 않음 |
| 작업 목록 | 진행 중인 작업 카드를 세로 스크롤 목록으로 표시 | 최신 접수 순 정렬 |
| 하단 네비게이션 | 5개 탭으로 주요 화면 이동 | 현재 탭(홈) 활성 상태 강조 |

## 3. 컴포넌트 목록

### 3.1 상단 앱바

| 컴포넌트 | 타입 | 설명 | 상태 |
|----------|------|------|------|
| 앱 로고 | Image/Text | "거트알림" 앱 타이틀 | 항상 표시 |
| 알림 아이콘 | IconButton | 알림 목록 화면으로 이동 | 읽지 않은 알림 있을 때 뱃지 숫자 표시 |

### 3.2 진행 중 요약 카드

| 컴포넌트 | 타입 | 설명 | 상태 |
|----------|------|------|------|
| 요약 카드 컨테이너 | Card | 진행 중 작업 건수 요약 | 작업이 0건이면 숨김 |
| 접수 건수 | Text | "접수 N건" | N이 0이면 "0건" 표시 |
| 작업중 건수 | Text | "작업중 N건" | N이 0이면 "0건" 표시 |

### 3.3 작업 목록

| 컴포넌트 | 타입 | 설명 | 상태 |
|----------|------|------|------|
| 작업 카드 | Card (반복) | 개별 작업 정보 표시 | 탭 가능 |
| 샵 이름 | Text | 작업을 맡긴 샵 이름 | 항상 표시 |
| 거트 이름 | Text | 사용 거트명 | 항상 표시 |
| 텐션 | Text | 설정 텐션값 | 항상 표시 |
| 라켓 모델 | Text | 라켓 모델명 | 항상 표시 |
| 상태 뱃지 | StatusBadge | 접수됨/작업중/완료 상태 표시 | 상태별 색상 구분 |
| 접수 시간 | Text | 접수 일시 | "N시간 전" 또는 "MM/DD HH:mm" 형식 |

### 3.4 상태 뱃지 색상

| 상태 | 색상 | 텍스트 |
|------|------|--------|
| `received` | 주황색 (`#FF9800`) | 접수됨 |
| `in_progress` | 파란색 (`#2196F3`) | 작업중 |
| `completed` | 초록색 (`#4CAF50`) | 완료 |

### 3.5 하단 네비게이션

| 컴포넌트 | 타입 | 설명 | 상태 |
|----------|------|------|------|
| 홈 탭 | BottomNavItem | 현재 화면 | 활성 (강조 색상) |
| 샵검색 탭 | BottomNavItem | 주변 샵 검색 화면으로 이동 | 비활성 |
| QR 탭 | BottomNavItem | 내 QR코드 화면으로 이동 | 비활성 |
| 이력 탭 | BottomNavItem | 작업 이력 화면으로 이동 | 비활성 |
| MY 탭 | BottomNavItem | 마이페이지로 이동 | 비활성 |

## 4. 필드 정의

이 화면은 입력 필드가 없는 조회 전용 화면이다. 표시되는 데이터 필드는 아래와 같다.

| 필드명 | 타입 | 데이터 출처 | 표시 형식 | 설명 |
|--------|------|-------------|-----------|------|
| shop_name | String | shops.name | 텍스트 | 샵 이름 |
| gut_name | String | orders.gut_name | 텍스트 | 거트명 |
| tension | String | orders.tension | 텍스트 (예: "25/27") | 텐션값 |
| racket_model | String | orders.racket_model | 텍스트 | 라켓 모델명 |
| status | Enum | orders.status | StatusBadge | 작업 상태 |
| created_at | Timestamp | orders.created_at | 상대시간 또는 날짜 | 접수 일시 |

## 5. 사용자 인터랙션

| # | 이벤트 | 트리거 | 액션 | 결과 |
|---|--------|--------|------|------|
| 1 | 작업 카드 탭 | 작업 카드 터치 | 작업 상세 화면으로 이동 | `customer-order-detail` 화면으로 전환, `order_id` 전달 |
| 2 | 알림 아이콘 탭 | 알림 아이콘 터치 | 알림 목록 화면으로 이동 | 알림 목록 화면 표시 |
| 3 | 하단 탭 전환 | 하단 네비게이션 탭 터치 | 해당 화면으로 이동 | 선택된 탭 화면으로 전환 |
| 4 | 아래로 당기기 | Pull-to-refresh 제스처 | 작업 목록 재조회 | 로딩 인디케이터 표시 후 목록 갱신 |
| 5 | 완료 작업 카드 탭 | 완료 상태 작업 카드 터치 | 작업 상세로 이동 (길찾기 버튼 포함) | `customer-order-detail` 화면, 길찾기 버튼 활성 |

## 6. API 연동

### 6.1 데이터 조회

| API | 테이블 | 조건 | 반환 데이터 | 호출 시점 |
|-----|--------|------|-------------|-----------|
| 내 작업 목록 조회 | `orders` JOIN `members` JOIN `shops` | `members.user_id = 현재_사용자_id` AND `orders.status IN ('received', 'in_progress', 'completed')` | order.*, shop.name | 화면 진입 시, Pull-to-refresh 시 |
| 진행 중 건수 조회 | `orders` JOIN `members` | `members.user_id = 현재_사용자_id` AND `orders.status IN ('received', 'in_progress')` | COUNT by status | 화면 진입 시 |

**Supabase 쿼리 예시:**

```dart
final orders = await supabase
  .from('orders')
  .select('*, members!inner(user_id, shop_id), shops!inner(name)')
  .eq('members.user_id', currentUserId)
  .in_('status', ['received', 'in_progress', 'completed'])
  .order('created_at', ascending: false);
```

### 6.2 데이터 변경

이 화면에서 데이터 변경은 발생하지 않는다.

### 6.3 실시간 구독

| 구독 대상 | 테이블 | 조건 | 이벤트 | 처리 |
|-----------|--------|------|--------|------|
| 내 작업 상태 변경 | `orders` | `members.user_id = 현재_사용자_id` | UPDATE (status 변경) | 해당 작업 카드의 상태 뱃지 즉시 갱신 |
| 새 작업 접수 | `orders` | `members.user_id = 현재_사용자_id` | INSERT | 작업 목록에 새 카드 추가 |

## 7. 상태 관리

### 7.1 화면 상태

| 상태 | 조건 | UI 표시 |
|------|------|---------|
| **로딩** | 화면 최초 진입 시 데이터 로드 중 | 화면 중앙에 CircularProgressIndicator 표시 |
| **빈 상태** | 조회 결과 작업이 0건 | 일러스트 + "아직 진행 중인 작업이 없습니다" + "주변 샵을 검색해보세요" 안내 텍스트 + 샵 검색 바로가기 버튼 |
| **에러** | 네트워크 오류 또는 서버 오류 | "데이터를 불러올 수 없습니다" + 재시도 버튼 |
| **정상** | 작업이 1건 이상 존재 | 요약 카드 + 작업 목록 표시 |

### 7.2 로컬 상태 변수

| 변수명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| `isLoading` | bool | true | 데이터 로딩 중 여부 |
| `orders` | List\<Order\> | [] | 조회된 작업 목록 |
| `receivedCount` | int | 0 | 접수 상태 작업 건수 |
| `inProgressCount` | int | 0 | 작업중 상태 작업 건수 |
| `hasError` | bool | false | 에러 발생 여부 |
| `errorMessage` | String | "" | 에러 메시지 |

## 8. 네비게이션

### 8.1 진입 경로

| 출발 화면 | 동작 | 전달 파라미터 |
|-----------|------|---------------|
| 로그인/회원가입 | 로그인 성공 후 자동 이동 | 없음 |
| 하단 네비게이션 | "홈" 탭 터치 | 없음 |
| 푸시 알림 | 알림 탭 시 앱 열림 | `order_id` (해당 작업 상세로 딥링크 가능) |

### 8.2 이동 가능 화면

| 대상 화면 | 동작 | 전달 파라미터 |
|-----------|------|---------------|
| 작업 상세 (`customer-order-detail`) | 작업 카드 탭 | `order_id` |
| 주변 샵 검색 (`customer-shop-search`) | 하단 "샵검색" 탭 또는 빈 상태 버튼 | 없음 |
| 내 QR코드 (`customer-my-qr`) | 하단 "QR" 탭 | 없음 |
| 작업 이력 (`customer-history`) | 하단 "이력" 탭 | 없음 |
| 마이페이지 (`customer-mypage`) | 하단 "MY" 탭 | 없음 |

## 9. 비고

- 완료된 작업은 홈 화면에 **24시간 동안만** 표시하고, 이후에는 "작업 이력" 화면에서만 확인 가능하도록 한다.
- 푸시 알림으로 앱 진입 시, `order_id`가 전달되면 홈을 거치지 않고 바로 작업 상세 화면으로 딥링크한다.
- 작업 카드에서 완료 상태인 경우, 카드 우측에 "길찾기" 아이콘 버튼을 추가로 표시하여 빠르게 샵 위치로 이동할 수 있게 한다.
- 실시간 구독은 앱이 포그라운드 상태일 때만 활성화하고, 백그라운드에서는 푸시 알림에 의존한다.
