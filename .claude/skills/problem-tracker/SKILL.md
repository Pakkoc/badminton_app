# Problem Tracker

사용자가 보고하는 오류를 `problem/problem_doc.md`에 체계적으로 기록하고, 관련 이미지를 관리한다.

## 존재 이유

오류를 일관된 형식으로 기록해야 추적과 해결이 쉽다. 이미지 증거를 오류와 함께 관리하면 디버깅 시 빠르게 맥락을 파악할 수 있다.

## 트리거

- "ㅇㅇㅇ 오류가 발생했습니다"
- "ㅇㅇㅇ 에러", "ㅇㅇㅇ 버그"
- "이런 문제가 있어요" + 설명
- 사용자가 스크린샷/이미지와 함께 오류를 보고할 때

---

## 프로토콜

### Step 1: 오류 정보 수집

사용자 메시지에서 다음을 파악한다:

| 항목 | 필수 | 설명 |
|------|------|------|
| **오류명** | ✅ | 짧고 명확한 오류 식별자 (예: `qr-scan-crash`, `login-timeout`) |
| **오류 설명** | ✅ | 사용자가 보고한 오류 내용 |
| **이미지** | ⬜ | 사용자가 첨부한 스크린샷/이미지 경로 |
| **재현 조건** | ⬜ | 오류 발생 조건 (있으면 기록) |

**오류명 규칙**: 사용자가 말한 오류 내용을 기반으로 `kebab-case`로 자동 생성한다.
- 예: "로그인 오류가 발생했습니다" → `login-error`
- 예: "QR 스캔이 안 됩니다" → `qr-scan-failure`

### Step 2: 이미지 처리 (이미지가 있을 때만)

1. 사용자가 제공한 이미지 파일 경로를 확인한다
2. `problem/img/` 폴더에 오류명에 맞는 이름으로 복사한다
   - 파일명 형식: `{오류명}.{확장자}` (예: `login-error.png`)
   - 같은 오류에 이미지가 여러 장이면: `{오류명}-1.png`, `{오류명}-2.png`
3. 원본 이미지는 그대로 유지한다 (삭제하지 않음)

```bash
# 예시
cp /path/to/user-image.png problem/img/login-error.png
```

### Step 3: problem_doc.md에 기록

`problem/problem_doc.md` 파일 끝에 아래 형식으로 추가한다:

```markdown
## {오류명}

- **날짜**: {YYYY-MM-DD}
- **상태**: 미해결
- **설명**: {사용자가 보고한 오류 내용}
- **재현 조건**: {있으면 기록, 없으면 생략}

{이미지가 있을 때만 아래 추가}
### 스크린샷
![{오류명}](img/{파일명})
```

**기록 규칙**:
- 파일이 비어있으면 최상단에 `# 오류 기록` 헤더를 먼저 추가한다
- 기존 내용이 있으면 파일 끝에 `---` 구분선 후 추가한다
- 날짜는 현재 날짜를 사용한다 (절대 날짜, 상대 날짜 아님)

### Step 4: 확인

사용자에게 기록 완료를 알린다:
- 기록된 오류명
- 이미지 저장 여부
- `problem/problem_doc.md` 경로
