# Common Modules — 공통 모듈 설계

## 개요

페이지/화면별 병렬 개발을 시작하기 전에, 공통으로 사용될 모듈과 라이브러리 설정을 설계한다. 이 문서가 완성되어야 이후 화면별 개발을 충돌 없이 병렬로 진행할 수 있다.

**시작 시 안내:** "common-modules 스킬을 실행합니다. 병렬 개발을 위한 공통 모듈을 설계합니다."

<HARD-GATE>
이 문서가 완성되기 전에 화면별 구현을 시작하지 않는다. 공통 모듈이 정의되지 않은 상태에서 병렬 개발하면 코드 충돌이 발생한다.
</HARD-GATE>

## 사용법

```
/common-modules           # 공통 모듈 설계 문서 생성
/common-modules update    # 기존 문서 갱신
```

## 실행 절차

### Step 1: 프로젝트 문서 파악

1. **전체 문서 읽기**:
   - `docs/service-overview*`, `docs/requirements*`
   - `docs/techstack*`
   - `docs/database*`
   - `docs/usecases/**/*.md`
2. **현재 코드베이스 분석** — 이미 구현된 코드가 있으면 구조를 파악

### Step 2: 공통 모듈 도출

모든 유스케이스를 교차 분석하여 2개 이상의 화면/기능에서 사용되는 공통 요소를 도출한다.

**일반적인 공통 모듈 카테고리**:

| 카테고리 | 예시 |
|---------|------|
| 초기화/설정 | BaaS 클라이언트, 환경 변수, 앱 설정 |
| 인증 | 로그인 상태 관리, 토큰 관리, 권한 체크 |
| 라우팅 | 라우터 설정, 가드/미들웨어 |
| 에러 처리 | 공통 에러 클래스, 에러 핸들러, 에러 UI |
| 네트워크 | API 클라이언트, 인터셉터, 캐시 |
| 공통 모델 | 엔티티, enum, DTO |
| 공통 위젯/컴포넌트 | 상태 뱃지, 빈 상태, 로딩, 토스트 |
| 외부 서비스 | 푸시 알림, 지도, 결제 등 래퍼 |
| 유틸리티 | 날짜 포맷, 전화번호 포맷, 유효성 검증 |
| 테스트 환경 | 테스트 설정, Mock, Fixture |

### Step 3: 모듈별 상세 설계

각 모듈에 대해 다음을 정의한다:

```markdown
### [모듈명]

**역할**: [이 모듈이 하는 일]
**파일 위치**: [예상 파일 경로]
**의존성**: [필요한 패키지/라이브러리]
**공개 인터페이스**:
  - [함수/클래스 시그니처]
  - [함수/클래스 시그니처]
**사용하는 화면**: [어느 화면에서 사용되는지]
```

### Step 4: 자가 검증 (3회 반복)

다음 질문에 모두 "예"로 답할 수 있어야 한다:

```
검증 1: 완전성
  - 모든 유스케이스에서 공통으로 쓰이는 코드가 빠짐없이 포함되었는가?
  - 2개 이상 화면에서 동일한 로직이 필요한 부분이 누락되지 않았는가?

검증 2: 독립성
  - 이 공통 모듈만 먼저 구현하면, 이후 모든 화면을 병렬로 개발할 수 있는가?
  - 화면 A 개발자가 화면 B 개발자의 코드를 기다려야 하는 상황이 없는가?

검증 3: 최소성
  - 과잉 설계된 모듈이 없는가?
  - 문서에 언급되지 않은 추측성 모듈이 포함되지 않았는가?
```

**3회 검증을 모두 통과해야 문서를 완성한다.**

### Step 5: 테스트 환경 설계

TDD 가이드라인에 따라 테스트 환경을 설계한다:
- 단위 테스트 프레임워크 설정
- Mock/Stub 전략
- 테스트 디렉토리 구조
- CI/CD 연동 방안

### Step 6: 문서 생성 및 커밋

`docs/common-modules.md`에 통합하여 생성하고 커밋한다.

## 프로젝트 탐색 규칙

이 스킬은 특정 프로젝트에 종속되지 않는다.

| 대상 | 탐색 방법 |
|------|----------|
| 기획 문서 | `docs/service-overview*`, `docs/requirements*` |
| 기술 스택 | `docs/techstack*` |
| DB 스키마 | `docs/database*` |
| 유스케이스 | `docs/usecases/**/*.md` |
| 기존 공통 모듈 | `docs/common-modules*` |
| 코드베이스 | `lib/`, `src/`, `app/` 등 |

## 핵심 원칙

1. **동기화 장벽** — 이 문서 완성 후에만 화면별 병렬 개발을 시작한다
2. **3회 자가 검증** — 완전성, 독립성, 최소성을 3회 반복 검증한다
3. **과잉 설계 금지** — 문서에 언급된 내용만으로 설계한다
4. **프로젝트 무관** — 어떤 프로젝트에서도 동일하게 동작한다
