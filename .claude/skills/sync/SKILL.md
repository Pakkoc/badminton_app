# Sync — 화면 레지스트리 동기화

## 개요

`docs/screen-registry.yaml`을 단일 진실 공급원(SOT)으로 사용하여 모든 산출물의 정합성을 검사하고 자동 수정한다.

**시작 시 안내:** "sync 스킬을 실행합니다. screen-registry.yaml 기준으로 전체 정합성을 검사합니다."

## 사용법

```
/sync              # 전체 검사 + 자동 수정
/sync check        # 검사만 (수정하지 않고 불일치 보고)
/sync [screen-id]  # 특정 화면만 검사 + 수정
```

## SOT 파일

- **위치**: `docs/screen-registry.yaml`
- **역할**: 모든 화면의 메타 정보 (ID, 이름, 역할, 네비게이션, Pencil ID, 스펙 파일 경로)를 한 곳에서 관리
- **변경 규칙**: 화면 추가/변경/삭제 시 **이 파일을 먼저 수정**한 후 `/sync`를 실행한다

## 검사 항목

### 1. 스펙 파일 존재 확인
- 레지스트리의 `spec_file` 경로에 실제 파일이 존재하는지 확인
- 없으면: 경고 출력

### 2. 화면 ID/이름 일치 확인
- 각 스펙 파일의 `화면 ID`와 `화면 명`이 레지스트리와 일치하는지 확인
- 불일치 시: 스펙 파일의 값을 레지스트리 기준으로 수정

### 3. Cross-reference 정합성
- 레지스트리의 `nav_to`에 있는 화면이 대상 화면의 `nav_from`에도 있는지 **양방향 확인**
- 각 스펙 파일의 네비게이션 섹션 (8.1, 8.2)에서 참조하는 화면 ID가 레지스트리에 존재하는지 확인
- 스펙 파일에서 사용하는 화면 ID/이름이 레지스트리와 일치하는지 확인
- 불일치 시: 레지스트리 기준으로 스펙 파일 수정

### 4. 역할(role) 일치 확인
- 스펙 파일의 `사용자 역할`이 레지스트리의 `role`과 일치하는지 확인

### 5. Pencil 디자인 확인 (선택)
- 레지스트리의 `pencil_id`가 실제 Pencil 파일에 존재하는지 확인
- Pencil MCP 도구가 사용 가능할 때만 실행

## 실행 절차

1. **레지스트리 읽기**: `docs/screen-registry.yaml`을 파싱한다
2. **전체 스펙 파일 스캔**: `docs/ui-specs/*.md`의 모든 파일을 읽는다
3. **검사 수행**: 위 5가지 검사 항목을 순서대로 실행한다
4. **결과 보고**: 불일치 항목을 테이블로 출력한다

```
## 검사 결과

| # | 파일 | 항목 | 현재값 | 기대값 | 상태 |
|---|------|------|--------|--------|------|
| 1 | login.md | 화면 명 | 로그인 | 로그인 | ✅ |
| 2 | shop-signup.md | nav_from | signup | profile-setup | ❌ 수정됨 |
```

5. **자동 수정** (`/sync` 또는 `/sync [screen-id]`):
   - 불일치 항목을 레지스트리 기준으로 자동 수정
   - 수정된 파일 목록 출력
   - 커밋 (메시지: `fix: screen-registry 기준 스펙 동기화`)

6. **검사만** (`/sync check`):
   - 불일치 항목을 보고만 하고 수정하지 않음

## 레지스트리 변경 시 워크플로우

```
화면 변경 발생 (예: signup → profile-setup)
  ↓
1. screen-registry.yaml 수정 (SOT 먼저!)
  ↓
2. /sync 실행
  ↓
3. 자동으로:
   - 해당 화면 스펙 파일의 ID/이름 수정
   - 다른 스펙에서 이 화면을 참조하는 부분 수정
   - 네비게이션 양방향 정합성 수정
  ↓
4. 결과 보고 + 커밋
```

## 화면 추가/삭제 시

### 화면 추가
1. `screen-registry.yaml`에 새 화면 정보 추가
2. `/sync` 실행 → 스펙 파일 미존재 경고
3. `/ui-screen-spec [screen-id]`로 스펙 생성
4. `/sync` 다시 실행 → 정합성 확인

### 화면 삭제
1. `screen-registry.yaml`에서 해당 화면 제거
2. `/sync` 실행 → 다른 스펙에서 삭제된 화면을 참조하는 부분 경고
3. 참조하는 스펙을 수동 수정 (삭제된 화면으로의 네비게이션 제거)
4. 스펙 파일 삭제
5. `/sync` 다시 실행 → 정합성 확인

## 핵심 원칙

1. **레지스트리가 항상 우선** — 레지스트리와 스펙이 다르면, 레지스트리가 정답이다
2. **양방향 정합성** — A→B 네비게이션이 있으면, B의 진입 경로에도 A가 있어야 한다
3. **변경은 SOT부터** — 어떤 변경이든 레지스트리를 먼저 수정하고, /sync로 전파한다
4. **검사 후 수정** — 자동 수정 전에 항상 불일치 목록을 먼저 보여준다
